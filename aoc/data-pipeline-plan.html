This document is the complete plan and requirements specification for rebuilding AJ Harbinger's Art of Charm (AOC) X-Factor Accelerator (XFA) landing page, powered by a data pipeline that mines sales call transcripts for real buyer language.

The pipeline runs on AJ's Mac (Apple M2, 16GB). Transcription uses Whisper locally + Fireflies API. Extraction uses Claude API. All scripts live in /Users/ajharbinger/Kas/scripts/.

Automated
  • • Unzip 4 archives into audio/zips/, extract only media files

  • • Model: openai/whisper-medium

  • • Process order: zips first, then Dropbox

Phase 3: Persona & Pain Extraction (Claude API)

Core Engine

scripts/extract_personas.py processes each transcript through Claude API. One JSON file per transcript in analysis/raw-extractions/.

  • Caller Profile Age range, gender, profession, life stage, relationship status
  • Pain Points Exact quotes, categorized practical vs emotional
  • Desired Outcomes Their words, practical vs emotional
  • Objections What held them back, why they almost didn't buy
  • Language Patterns Exact phrases describing their problem
  • Trigger Event What made them book NOW vs 6 months ago
  • Previous Attempts Books, therapy, other programs they've tried
  • Identity Language "I'm an introvert," "I'm socially awkward"
  • Buying Signals What made them say yes, what tipped them over
  • Price Sensitivity Cost mentions, affordability, value justification
  • Urgency Indicators Deadlines, life events, career pressure
  • Discovery Source How they found AOC: podcast, referral, Google, etc.

Phase 4: Consolidation & Filtering

Analysis
  • • Merge all extractions into master lists
  • • Deduplicate similar personas into archetypes
  • • Rank pain points by frequency
  • • Rank outcomes by intensity
  • • Rank trigger events by urgency (higher urgency = better ad targeting)
  • • Map previous attempts to positioning angles ("unlike therapy/books/courses...")
  • • Apply Kas's filtering lens (see below)

Phase 4.5: Market Demand Validation

Gate Check

Before finalizing deliverables, validate top pain points against real search demand. This is the "would someone Google this?" test with actual data.

  • • Run top 20 pain point phrases through Google Ads Keyword Planner (search volume, CPC, competition)
  • • Check Google Trends for seasonality and trajectory
  • • Cross-reference with Reddit/Quora threads for raw language and thread engagement
  • • Score each pain point: search volume x buyer intent x AOC's ability to solve = priority rank
  • • Kill any persona/pain that fails the "can we run a Google ad for it?" test
  • • Output: analysis/market-validation.md with scored rankings

Kas's Filtering Lens

Applied in Phase 4

Every pain point and outcome gets scored against these 7 criteria. Fail any = deprioritize.

  • 1 Would someone Google this specific problem/outcome?
  • 2 Would they spend money TODAY to solve it?
  • 3 Is the total addressable market big enough?
  • 4 Can we build a lead magnet around it?
  • 5 Can we run a Google ad for it?
  • 6 Does it differentiate AOC from therapy, books, and generic "confidence" programs?
  • 7 Does it match the buyer profile? (39yo, $10K+/mo, analytical, 87% relationship-motivated)

Behavioral Psychology Layer (Rory Sutherland Frameworks)

LP Build

Applied during extraction analysis and LP copywriting. Reference: rory-sutherland-marketing-applications.md for full model-to-CRO mapping.

Reframing
Each pain point reframed from "I'm broken" to "I'm missing a skill." Reduces shame, increases action.
Psychological Moonshots
Small intervention, disproportionate perceived value. E.g., AI personality profile feels like $500 of therapy.
Red Bus / Blue Bus
Where is AOC competing in crowded "self-help" vs creating a new category entirely?
Satisficing vs Maximizing
XFA buyer is a maximizer (analytical, high-income). LP copy must give them enough data for due diligence.
Real Scarcity
Not fake countdown timers. Actual cohort limits, real enrollment windows, genuine capacity constraints.
Social Proof Architecture
11,700 alumni, 18 years, 1,000+ podcast episodes. Massive proof that needs to be structured, not dumped.
