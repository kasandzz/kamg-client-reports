1. Stripe API Key

~3 minutes
Why we need it: Ground truth for all payments -- $27 workshop tickets, $54 VIP upsells, and high-ticket enrollments. We only need read-only access (restricted key). Nothing will be charged or modified.

Go to the Stripe API Keys page

➔ https://dashboard.stripe.com/apikeys

Log in with the COD Stripe account. Make sure you're in Live mode (not Test) -- there's a toggle at the top of the page.

Click "Create restricted key"

Do NOT use the default secret key. We want a restricted key so it can only read, not charge.

2. Google Analytics 4 (GA4)

~5 minutes
Why we need it: Traffic sources, landing page performance, user behavior. Backup to PostHog. We need the Property ID + viewer access for a service account.

Go to GA4 Admin

➔ https://analytics.google.com/analytics/web/

Log in with the COD Google account. Click the gear icon (Admin) in the bottom-left.

Find the Property ID

In Admin, look at the Property column. The Property ID is at the top -- it's a number like 123456789.

┌─────────────────────────────────────┐ │ Admin │ │ │ │ Account Property │ │ COD clientsondemand.com │ │ Property ID: 123456789 │ │ ▲▲▲▲▲▲▲▲▲ │ │ COPY THIS │ └─────────────────────────────────────┘

Also note the Property name so we know which site it's for.

Add our service account as a Viewer

Still in Admin, go to Property Access Management (under the Property column).

➔ Admin > Property > Property Access Management

Click the blue + button in the top right, then "Add users".

Enter this email address:

Service Account Email claude-sync@kasandz-automations.iam.gserviceaccount.com

Set the role to Viewer (the lowest permission -- read-only).

Uncheck "Notify new users by email" (it's a service account, not a person).

Click Add.

💡 If COD has multiple GA4 properties (e.g., one for clientsondemand.com, one for clientsondemand.live), we need the Property IDs for ALL of them. Just send Kas all the IDs you see.

What to send Kas

~5 minutes
Why we need it: Landing page behavior -- scroll depth, button clicks, time on page. Matt set it up via GTM (M677PM4) but we need to verify it's working AND get an API key to pull data.

Log into PostHog

➔ https://app.posthog.com

If you don't have access, check with Matt Freeman -- he set up the PostHog project.

First: verify events are flowing

Go to Activity > Live Events (left sidebar). You should see a live stream of pageviews and events coming in from COD's sites.

┌──────────────────────────────────────────┐ │ Live Events │ │ │ │ 12:01 PM $pageview clientsondemand.live/... │ │ 12:01 PM $autocapture button click │ │ 12:00 PM $pageview clientsondemand.com/... │ │ │ │ If you see events = IT'S WORKING │ │ If blank/empty = IT'S NOT SET UP YET │ └──────────────────────────────────────────┘

Take a quick screenshot of this page either way and send it to Kas.

Get the Project API Key

Go to Settings (gear icon, bottom-left) then Project > Project API Key.

➔ https://app.posthog.com/settings/project#variables

Copy the Project API Key (starts with phc_).

Create a Personal API Key (for querying)

The Project API Key only works for ingesting events. To read data, we need a Personal API Key:

➔ https://app.posthog.com/settings/user#personal-api-keys

Go to Settings > User > Personal API Keys.

Click "Create personal API key".

Name: KAMG Data Warehouse

Scopes: select Read for everything (or "All read" if there's a shortcut).

Copy the key (starts with phx_). You can only see it once!

What to send Kas

1. Screenshot of Live Events page (working or not)
2. Project API Key: phc_xxxxx
3. Personal API Key: phx_xxxxx

5. Meta Ads API (Facebook/Instagram)

~10 minutes
Why we need it: This is the #1 blocker. Without Meta spend data, we literally cannot calculate true ROAS (revenue per dollar spent per creative). We need a System User token from Business Manager -- this is the secure, long-lived way to pull ad data.
⚠ This one requires Business Manager Admin access. If you don't have BM admin, Akari or Pedro will need to do this step (or grant you admin).

Go to Meta Business Settings

➔ https://business.facebook.com/settings

Log in with the account that has admin access to COD's Business Manager.

Navigate to System Users

In the left sidebar: Users > System Users

┌─────────────────────────────────────┐ │ Business Settings │ │ │ │ Users │ │ People │ │ Partners │ │ System Users ◄◄◄ CLICK │ └─────────────────────────────────────┘

Create a new System User

Click "Add" button.

Name: KAMG Data Pipeline

Role: Admin (needed to read ad data across all campaigns)

Click "Create System User".

Assign the Ad Account

After creating the system user, click on it, then click "Add Assets".

Select Ad Accounts from the left, find COD's ad account, and check it.

Toggle permissions: enable "View performance" (we don't need "Manage campaigns").

┌────────────────────────────────────────────┐ │ Add Assets to: KAMG Data Pipeline │ │ │ │ Ad Accounts | Pages | Pixels │ │ │ │ [x] COD Ad Account (act_XXXXXXXXX) │ │ │ │ Permissions: │ │ [x] View performance ◄◄◄ CHECK THIS │ │ [ ] Manage campaigns │ │ │ │ [Save Changes] │ └────────────────────────────────────────────┘

Click "Save Changes".

Generate the Access Token

Click on the system user again, then click "Generate New Token".

Select the app (if there's no app, you may need to create one first -- see info box below).

Token permissions needed:

Required Permissions ads_read ads_management (needed for reading insights, despite the name) read_insights business_management

Token expiration: choose "Never" if available, otherwise the longest option.

Click "Generate Token". Copy it immediately!

⚠ This token is shown ONCE. Copy and send to Kas right away via DM.

Also grab the Ad Account ID

➔ https://business.facebook.com/settings/ad-accounts

Click on COD's ad account. The ID is a number like act_123456789. Copy it.

💡 No app exists? If the "Generate Token" step asks for an app and there isn't one:
1. Go to developers.facebook.com/apps
2. Click "Create App" > "Other" > "Business"
3. Name it "COD Data Pipeline", link it to COD's Business Manager
4. No need to configure anything -- just having the app lets you generate tokens
5. Then come back and generate the token

What to send Kas

1. System User Access Token: EAAG... (long string)
2. Ad Account ID: act_123456789
3. Business Manager ID (shown at top of Business Settings)
