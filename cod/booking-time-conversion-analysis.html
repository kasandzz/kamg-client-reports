The Real Answer
Same-day afternoon actually has the BEST show rate (100%) -- zero no-shows when someone books and shows up the same day. The previous report was misleading because "Rescheduled" in the Google Sheet was treated as a negative signal, but Hyros shows those rescheduled prospects eventually showed up as QUALIFIED calls.

Next-day morning is still excellent at 94.3%, but has a small no-show rate (5.7%). The real enemy isn't the time slot -- it's the 2-4 day gap where 9% no-show and 18% cancel.

Key Insight: The Problem Is Cancellation, Not No-Shows
Show rates (of those who don't cancel) are strong across all segments (91-100%). The real lever is cancellation rate: same-day slots cancel 14-16%, while 2-4 day and 5+ day slots cancel 18-23%. Reducing the booking window from 5+ days to 2-3 days max would cut cancellations.

Methodology & Data Sources

How "Showed" Was Determined
  • Source: GHL Calendar API appointments (fact_bookings) joined to ghl_contacts (for email) joined to Hyros call records
  • Join: fact_bookings.contact_id -> ghl_contacts.contact_id -> hyros_calls.email (lowercase trimmed match)
  • Logic: If Hyros has a QUALIFIED call for that email, the booking is marked "showed." NO_SHOW and CANCELLED from Hyros override GHL status.
  • Dedup: When a contact has multiple Hyros records, the most recent call_state is used (ROW_NUMBER by call_timestamp DESC)
Booking Lag (Momentum Segment) Source
  • Source: sheets_bookings (Google Sheets tracker). This table has separate date_booked (when they clicked book) and call_date (scheduled call datetime as a formatted string).
  • Why not fact_bookings: GHL Calendar API doesn't return createdAt, so date_booked falls back to startTime (appointment time). All records show 0-day lag. Booking lag can only be calculated from sheets_bookings.
  • Timezone: Hourly analysis uses fact_bookings time_booked (UTC, converted to ET with -5h offset). Call_date strings in sheets_bookings are assumed ET.
Remaining Gaps
  • Enrollment: Still not trackable. Stripe only has $27 tickets (7 email matches to bookings). High-ticket enrollment goes through Authorize.net/separate Stripe. Need that data source connected.
  • Unresolved records: 204 bookings (15%) have no Hyros match. Could be test records, different email used, or calls that happened outside Hyros tracking window.
  • EDT vs EST: UTC-5 used for all months. Apr-Oct should technically be UTC-4 (EDT). This shifts hours by 1 during daylight saving.

Recommended Actions

P0

Reduce 1-3 PM No-Show Rate

The 1-3 PM ET window has 10-15% no-show rates vs 1-5% for morning and evening slots. Add aggressive SMS confirmations (2hr + 30min before) specifically for afternoon bookings. Consider reducing afternoon slot availability.

P0

Connect Enrollment Data

Show rate question is answered. The next question is: which time slots produce enrollments? Need Authorize.net or high-ticket Stripe account data in the warehouse, joinable by email.

P1

Cap Booking Window to 3 Days

5+ days out has 23.3% cancel rate (vs 14% same-day). Tighter booking windows reduce cancellation. Consider max 3-day calendar window with waitlist for overflow.

P1

Favor 7 PM + 12 PM Slots

These two hours account for 75% of all bookings AND have the best show rates (97-99%). If expanding closer capacity, prioritize adding slots at these times.

P2

Fix GHL ETL: Add createdAt

The GHL Calendar API may support createdAt on appointment objects. If so, update etl-ghl-calendar.js to capture it. This would let us calculate booking lag directly from fact_bookings instead of depending on sheets_bookings.
