Share Facebook API Token with KAMG

Hi Akari! Follow the steps below. Takes about 3 minutes.

Step 1: Open System Users

  1. Click the button below to open Meta Business Settings
    Open System Users
  2. Click Add to create a new system user
    Name it something like "KAMG API Access"
  3. Set the role to Admin
    This ensures the token can access everything we need

Step 2: Assign Assets

  1. Click on the system user you just created
  2. Click Add Assets
  3. Under Ad Accounts, select ALL ad accounts and set to Full Control
  4. Under Pages, select ALL pages and set to Full Control
  5. Under Pixels, select ALL pixels and set to Full Control
  6. If you see Catalogs, Apps, or Instagram Accounts, add those too with Full Control

Step 3: Generate the Token

  1. Click Generate New Token
  2. Select your app from the dropdown
    If no app exists, you may need to create one here first
  3. Set token expiration to Never (60-day tokens break automations)
  4. Check ALL of the following permissions:

    Ads & Campaigns

    • ads_read - Read ad account data, campaigns, ad sets, ads
    • ads_management - Manage and create campaigns
    • business_management - Access business-level settings and assets

    Insights & Analytics

    • read_insights - Ad performance metrics (spend, ROAS, CPA, etc.)
    • pages_read_engagement - Page post engagement data
    • pages_read_user_content - User comments and posts on pages

    Pages & Content

    • pages_manage_posts - Read and manage page posts
    • pages_manage_metadata - Page settings and metadata
    • pages_manage_engagement - Manage comments and messages
    • pages_manage_ads - Manage ads linked to pages
    • pages_show_list - List all pages the user manages

    Leads & Conversions

    • leads_retrieval - Pull lead form submissions
    • page_events - Conversion events on pages

    Instagram

    • instagram_basic - Basic Instagram account info
    • instagram_manage_insights - Instagram analytics and metrics
    • instagram_manage_comments - Read and manage IG comments
    • instagram_content_publish - Publish content to Instagram

    Audiences & Catalog

    • catalog_management - Product catalogs for dynamic ads
    • custom_audience - Custom and lookalike audiences

    Messaging

    • pages_messaging - Read page Messenger conversations

    Pixel & Tracking

    • ads_read - (already checked above) includes pixel data
    • business_management - (already checked above) includes pixel management
  5. Click Generate Token

Step 4: Send the Token to Kas

  1. Copy the token immediately - you won't be able to see it again!
  2. Send it to Kas via direct message only
    Slack DM, text, or email - never in a group channel
Important: This token is like a password. Only share it directly with Kas. If it ever gets exposed, you can revoke it from the same System Users page and generate a new one.

That's it!

Once Kas has the token, KAMG can pull your ad data, insights, leads, and audience info for reporting and optimization. No further action needed from you.
