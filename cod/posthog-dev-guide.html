Why Only 2 Snippets Are Essential

Hyros already tracks conversion events (ticket purchase, call booked, enrollment). PostHog's unique value is on-page behavioral data that Hyros can't see: heatmaps, session recordings, scroll depth, and workshop watch time.

For PostHog's behavioral data to be useful in the data warehouse, we need exactly two things:

  1. Link anonymous visitors to email addresses (so PostHog sessions can join to Hyros/GHL data)
  2. Track workshop watch duration (PostHog's biggest blind-spot filler, since no other tool tracks this)

Everything else (ticket purchase events, call booked events, etc.) is already covered by Hyros and only adds marginal value inside PostHog's own UI.

How it works:

  1. GTM loads the PostHog SDK on every COD page.
  2. Snippets wait for window.posthog to be available, then fire page-specific events.
  3. GHL form submissions emit window.postMessage events containing form data. Snippet 1 listens for these.
  4. Events flow to PostHog Cloud, then batch export nightly to BigQuery.

GHL Installation Path

For every snippet below:

  1. Open GHL > Sites > Funnels (or Sites > Websites).
  2. Select the target funnel/page.
  3. Click Settings (gear icon) on the page.
  4. Navigate to Custom Code > Footer Code (sometimes labeled "Body Tracking Code" or "Footer").
  5. Paste the full <script> block.
  6. Click Save, then Publish the page.
Important: GHL changes are not live until you hit Publish. Always republish after pasting.

REQUIRED: Snippet 1 -- User Identification (Registration Pages)

Install on: Every registration and opt-in page, including:

  • clientsondemand.live/registration2
  • All segment-specific pages (therapists, attorneys, educators, coaches, health coaches)

What it does: Listens for GHL form submissions via postMessage, extracts the lead's email/name/phone, identifies them in PostHog, and fires a lead_registered event with UTM parameters. This is what links anonymous browser sessions to known customer emails in the data warehouse.

REQUIRED Snippet 1: User Identification
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.identify) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  window.addEventListener('message', function(event) {
    try {
      var data = typeof event.data === 'string' ? JSON.parse(event.data) : event.data;
      if (data && (data.type === 'form:submit' || data.event_id === 'form_submit')) {
        var email = data.email || data.data?.email || '';
        var firstName = data.first_name || data.data?.first_name || data.firstName || '';
        var lastName = data.last_name || data.data?.last_name || data.lastName || '';
        var phone = data.phone || data.data?.phone || '';
        if (email) {
          waitForPostHog(function() {
            posthog.identify(email, {
              email: email,
              first_name: firstName,
              last_name: lastName,
              phone: phone,
              stage: 'lead',
              source: 'registration_page',
              page_url: window.location.pathname
            });
            posthog.capture('lead_registered', {
              email: email,
              page_url: window.location.pathname,
              utm_source: new URLSearchParams(window.location.search).get('utm_source'),
              utm_medium: new URLSearchParams(window.location.search).get('utm_medium'),
              utm_campaign: new URLSearchParams(window.location.search).get('utm_campaign')
            });
          });
        }
      }
    } catch(e) {}
  });
})();
</script>
Verify: Submit a test registration. In PostHog, go to Activity > Live Events. You should see:
  • A $identify call with the email as distinct_id
  • A lead_registered event with UTM params and page_url

REQUIRED: Snippet 2 -- Workshop Watch Time (Workshop Page)

Install on: The workshop viewing page and any replay pages.

What it does: Fires workshop_attended on page load, then fires workshop_watch_milestone every 5 minutes with a running minutes_watched counter. Timer caps at 180 minutes (the workshop runs approximately 157 minutes). This is the only source of workshop engagement data in the entire COD stack.

REQUIRED Snippet 2: Workshop Watch Time
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.capture) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  waitForPostHog(function() {
    posthog.capture('workshop_attended', {
      page_url: window.location.pathname,
      timestamp: new Date().toISOString()
    });
    posthog.people.set({ stage: 'workshop_attendee' });
    var minutes = 0;
    var timer = setInterval(function() {
      minutes += 5;
      posthog.capture('workshop_watch_milestone', {
        minutes_watched: minutes,
        page_url: window.location.pathname
      });
      if (minutes >= 180) clearInterval(timer);
    }, 5 * 60 * 1000);
  });
})();
</script>
Verify: Load the workshop page. Confirm workshop_attended fires immediately. Wait 5 minutes and confirm a workshop_watch_milestone event appears with minutes_watched: 5.
Note: This snippet intentionally has no deduplication. Repeat visits and replays are valid engagement signals.

BigQuery Integration

Events batch-export daily to BigQuery.

Dataset: cod_warehouse
Table: posthog_events

Sample Queries

Workshop watch duration by user:

SELECT
  distinct_id AS email,
  MAX(CAST(JSON_EXTRACT_SCALAR(properties, '$.minutes_watched') AS INT64)) AS max_minutes,
  COUNT(*) AS milestones_hit
FROM `cod_warehouse.posthog_events`
WHERE event = 'workshop_watch_milestone'
  AND timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)
GROUP BY distinct_id
ORDER BY max_minutes DESC;

Join PostHog behavior to Hyros revenue:

SELECT
  ph.distinct_id AS email,
  MAX(ph_mins.max_minutes) AS workshop_minutes,
  hs.revenue,
  hs.firstSource
FROM `cod_warehouse.posthog_events` ph
LEFT JOIN (
  SELECT distinct_id, MAX(CAST(JSON_EXTRACT_SCALAR(properties, '$.minutes_watched') AS INT64)) AS max_minutes
  FROM `cod_warehouse.posthog_events`
  WHERE event = 'workshop_watch_milestone'
  GROUP BY distinct_id
) ph_mins ON ph.distinct_id = ph_mins.distinct_id
LEFT JOIN `cod_warehouse.hyros_sales` hs ON ph.distinct_id = hs.email
WHERE ph.event = 'lead_registered'
GROUP BY ph.distinct_id, hs.revenue, hs.firstSource
ORDER BY hs.revenue DESC NULLS LAST;

Notes for Rehan

  • The 2 required snippets are independent. Install in either order.
  • The waitForPostHog pattern handles race conditions where the snippet loads before GTM injects PostHog. The 500ms polling interval is a safe default.
  • If GHL changes their postMessage format, Snippet 1 will need updating. Monitor the browser console for message events to catch format changes early.
  • For segment-specific registration pages, Snippet 1 is identical across all of them. The page_url property automatically captures which segment page converted.
  • Optional snippets can be added anytime later with zero impact on the required ones.
