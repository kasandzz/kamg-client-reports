How it works

  1. GTM loads the PostHog SDK on every COD page.
  2. Each snippet waits for window.posthog to be available, then fires page-specific events.
  3. GHL form submissions emit window.postMessage events containing form data. Snippets 1 and 6 listen for these.
  4. sessionStorage keys prevent duplicate event fires on page refresh.
  5. Events flow to PostHog Cloud, then batch export nightly to BigQuery.

GHL Installation Path

For every snippet below, the installation steps are identical:

  1. Open GHL > Sites > Funnels (or Sites > Websites).
  2. Select the target funnel/page.
  3. Click Settings (gear icon) on the page.
  4. Navigate to Custom Code > Footer Code (sometimes labeled "Body Tracking Code" or "Footer").
  5. Paste the full <script> block.
  6. Click Save, then Publish the page.
Important: GHL changes are not live until you hit Publish. After pasting, always republish.

Snippet 1: User Identification (Registration Pages)

Install on: Every registration and opt-in page, including:

  • clientsondemand.live/registration2
  • All segment-specific pages (therapists, attorneys, educators, coaches, health coaches)

What it does: Listens for GHL form submissions via postMessage, extracts the lead's email/name/phone, identifies them in PostHog, and fires a lead_registered event with UTM parameters.

Snippet 1: User Identification Target: All registration/opt-in pages 
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.identify) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  window.addEventListener('message', function(event) {
    try {
      var data = typeof event.data === 'string' ? JSON.parse(event.data) : event.data;
      if (data && (data.type === 'form:submit' || data.event_id === 'form_submit')) {
        var email = data.email || data.data?.email || '';
        var firstName = data.first_name || data.data?.first_name || data.firstName || '';
        var lastName = data.last_name || data.data?.last_name || data.lastName || '';
        var phone = data.phone || data.data?.phone || '';
        if (email) {
          waitForPostHog(function() {
            posthog.identify(email, {
              email: email,
              first_name: firstName,
              last_name: lastName,
              phone: phone,
              stage: 'lead',
              source: 'registration_page',
              page_url: window.location.pathname
            });
            posthog.capture('lead_registered', {
              email: email,
              page_url: window.location.pathname,
              utm_source: new URLSearchParams(window.location.search).get('utm_source'),
              utm_medium: new URLSearchParams(window.location.search).get('utm_medium'),
              utm_campaign: new URLSearchParams(window.location.search).get('utm_campaign')
            });
          });
        }
      }
    } catch(e) {}
  });
})();
</script>
Verify
Submit a test registration. In PostHog, go to Activity > Live Events. You should see:
  • A $identify call with the email as distinct_id
  • A lead_registered event with UTM params and page_url

Snippet 2: $27 Ticket Purchase (Thank-You Page)

Install on: The post-purchase confirmation page shown after $27 workshop ticket checkout.

What it does: Fires a ticket_purchased event with revenue data and sets the person's stage to ticket_buyer. Uses sessionStorage to prevent duplicate fires on page refresh.

Snippet 2: Ticket Purchase Target: $27 thank-you/confirmation page 
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.capture) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  if (sessionStorage.getItem('ph_ticket_fired')) return;
  sessionStorage.setItem('ph_ticket_fired', '1');
  waitForPostHog(function() {
    posthog.capture('ticket_purchased', {
      amount: 27,
      currency: 'USD',
      product: 'workshop-ticket',
      product_type: 'workshop-$27',
      page_url: window.location.pathname
    });
    posthog.people.set({ stage: 'ticket_buyer' });
  });
})();
</script>
Verify
Complete a test purchase. In PostHog Live Events, confirm a ticket_purchased event appears with amount: 27. Refresh the page and confirm the event does NOT fire again.

Snippet 4: Workshop Attended (Workshop Page)

Install on: The workshop viewing page and any replay pages.

What it does: Fires workshop_attended on page load, then fires workshop_watch_milestone every 5 minutes with a running minutes_watched counter. Timer caps at 180 minutes (the workshop runs approximately 157 minutes). Sets stage to workshop_attendee.

Snippet 4: Workshop Attended Target: Workshop/replay page 
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.capture) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  waitForPostHog(function() {
    posthog.capture('workshop_attended', {
      page_url: window.location.pathname,
      timestamp: new Date().toISOString()
    });
    posthog.people.set({ stage: 'workshop_attendee' });
    var minutes = 0;
    var timer = setInterval(function() {
      minutes += 5;
      posthog.capture('workshop_watch_milestone', {
        minutes_watched: minutes,
        page_url: window.location.pathname
      });
      if (minutes >= 180) clearInterval(timer);
    }, 5 * 60 * 1000);
  });
})();
</script>
Verify
Load the workshop page. Confirm workshop_attended fires immediately. Wait 5 minutes and confirm a workshop_watch_milestone event appears with minutes_watched: 5.
Note: This snippet intentionally has no deduplication. Repeat visits and replays are valid engagement signals.

Snippet 6: Application Submitted (Apply Page)

Install on: clientsondemand.com/apply

What it does: Listens for GHL form submission via postMessage, identifies the user by email, and fires an application_submitted event. Sets stage to applicant.

Snippet 6: Application Submitted Target: clientsondemand.com/apply 
<script>
(function() {
  function waitForPostHog(cb) {
    if (window.posthog && window.posthog.capture) return cb();
    setTimeout(function() { waitForPostHog(cb); }, 500);
  }
  window.addEventListener('message', function(event) {
    try {
      var data = typeof event.data === 'string' ? JSON.parse(event.data) : event.data;
      if (data && (data.type === 'form:submit' || data.event_id === 'form_submit')) {
        var email = data.email || data.data?.email || '';
        waitForPostHog(function() {
          if (email) {
            posthog.identify(email, { stage: 'applicant' });
          }
          posthog.capture('application_submitted', {
            email: email,
            page_url: window.location.pathname
          });
        });
      }
    } catch(e) {}
  });
})();
</script>
Verify
Submit a test application. Confirm application_submitted and $identify events appear in Live Events.

BigQuery Integration

Events batch-export daily to BigQuery.

Dataset: cod_warehouse
Table: posthog_events

Sample Queries

Count leads registered by UTM source (last 7 days):

SQL: Leads by UTM Source cod_warehouse.posthog_events 
SELECT
  JSON_EXTRACT_SCALAR(properties, '$.utm_source') AS utm_source,
  COUNT(*) AS leads
FROM `cod_warehouse.posthog_events`
WHERE event = 'lead_registered'
  AND timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 7 DAY)
GROUP BY utm_source
ORDER BY leads DESC;

Funnel conversion rates:

SQL: Funnel Conversion Rates cod_warehouse.posthog_events 
WITH stages AS (
  SELECT
    distinct_id,
    MAX(CASE WHEN event = 'lead_registered' THEN 1 ELSE 0 END) AS registered,
    MAX(CASE WHEN event = 'ticket_purchased' THEN 1 ELSE 0 END) AS purchased,
    MAX(CASE WHEN event = 'workshop_attended' THEN 1 ELSE 0 END) AS attended,
    MAX(CASE WHEN event = 'call_booked' THEN 1 ELSE 0 END) AS booked
  FROM `cod_warehouse.posthog_events`
  WHERE timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)
  GROUP BY distinct_id
)
SELECT
  COUNT(*) AS total_leads,
  SUM(purchased) AS ticket_buyers,
  SUM(attended) AS workshop_attendees,
  SUM(booked) AS calls_booked,
  ROUND(SUM(purchased) / COUNT(*) * 100, 1) AS reg_to_purchase_pct,
  ROUND(SUM(booked) / NULLIF(SUM(attended), 0) * 100, 1) AS attend_to_book_pct
FROM stages
WHERE registered = 1;

VIP upgrade rate:

SQL: VIP Upgrade Rate cod_warehouse.posthog_events 
SELECT
  COUNTIF(event = 'vip_purchased') AS vip_buyers,
  COUNTIF(event = 'ticket_purchased') AS ticket_buyers,
  ROUND(COUNTIF(event = 'vip_purchased') / NULLIF(COUNTIF(event = 'ticket_purchased'), 0) * 100, 1) AS vip_rate_pct
FROM `cod_warehouse.posthog_events`
WHERE event IN ('ticket_purchased', 'vip_purchased')
  AND timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY);

Troubleshooting

Problem Cause Fix
posthog is not defined in console GTM container not firing on that page Verify GTM container M677PM4 is installed on the page. Check GHL global header code or GTM trigger conditions.
Snippet installed but no events in PostHog Page not published after edit Click Publish in GHL after saving footer code.
Events fire but no person identified Snippet 1 not on registration page, or form postMessage format changed Check browser console for postMessage payloads. GHL may have updated the event format.
Duplicate events on same page sessionStorage key not set (snippet error) Check browser console for JS errors in the snippet. Ensure the <script> tags are correct.
Events fire but missing UTM params UTM parameters stripped before page load Check that Meta Ads URLs pass UTMs. GHL may strip query params on redirect.
posthog.people.set is not a function PostHog SDK version mismatch Check GTM tag configuration. people.set requires PostHog JS SDK v1.x. For v2+, use posthog.setPersonProperties() instead.
Workshop milestones stop after page navigate Timer cleared on navigation (expected) This is correct behavior. Milestones only track active page time.
Form submit event not detected GHL iframe form isolation If GHL renders the form in an iframe, postMessage may not bubble to parent. Test with window.addEventListener('message', console.log) to confirm events arrive.

Notes for Rehan

  • All six snippets are independent. Install them in any order.
  • The waitForPostHog pattern handles race conditions where the snippet loads before GTM injects PostHog. The 500ms polling interval is a safe default.
  • Person properties use a progressive stage field. The last stage set wins. This is intentional for funnel analysis (a person's current stage reflects their furthest progress).
  • If GHL changes their postMessage format, Snippets 1 and 6 will need updating. Monitor the browser console for message events to catch format changes early.
  • For segment-specific registration pages, Snippet 1 is identical across all of them. The page_url property automatically captures which segment page converted.
