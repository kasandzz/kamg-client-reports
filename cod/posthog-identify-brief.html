COD / Dev Brief

PostHog Identify Integration

Link anonymous browsing behavior to real people
HIGH PRIORITY Effort: ~30 min 2 locations

The Problem

PostHog is tracking every click, pageview, and CTA interaction on the funnel pages. But right now every visitor is just an anonymous device ID like 0196a13e-7bfc-7f6c-.... We can see what happens on each page, but not who did it.

Result: We have 62,000+ events in our data warehouse but can't connect them to actual COD customers. One small code change fixes this.

The Fix

Add one posthog.identify() call when a form is submitted. PostHog will retroactively link all that person's anonymous browsing to their email. One line of code, two locations.

Location 1: Registration Page

Page: codworkshop.com (GHL form submit)

On the reg page form submit callback, add this before the redirect/thank-you:

// GHL form submit listener
document.querySelector('form').addEventListener('submit', function(e) {
  var email = document.querySelector('input[name="email"]').value;
  var firstName = document.querySelector('input[name="first_name"]').value;

  if (email) {
    posthog.identify(email.toLowerCase().trim(), {
      first_name: firstName,
      source: 'registration'
    });
  }
});
Note: Check the actual input field names in GHL -- they might be email, Email, or contact.email. Inspect the form HTML to confirm.

Location 2: Application / Thank You Page

Page: codworkshop.com/thankyou/

The SubmitApplication event already fires here. Add identify right before it:

// Before the existing SubmitApplication event
posthog.identify(email.toLowerCase().trim(), {
  source: 'application'
});

// Existing event (already working)
posthog.capture('SubmitApplication');

Fallback: Email from URL

Some GHL pages pre-fill email from URL params (like he=jennifer@example.com). If the form field isn't accessible, grab it from the URL:

// Fallback: grab email from URL if form field is empty
var urlParams = new URLSearchParams(window.location.search);
var email = urlParams.get('he') || urlParams.get('email');

if (email) {
  posthog.identify(email.toLowerCase().trim());
}

Important Rules

  • Use email as the ID (lowercased, trimmed) -- this matches our BigQuery warehouse key
  • posthog.identify() must be called after the PostHog snippet has loaded
  • PostHog will automatically merge all previous anonymous events with this person
  • This does NOT affect any existing tracking -- it only adds identity on top
