B1 Fix Mobile Form Disappearing Bug Critical

Application page (clientsondemand.live/application) New B-test form build
Why: Kas identified: when selecting any bottom options on the first question, the form disappears from the screen (shrinks). This is causing a significant portion of the 75% dropoff on the booking page. Mobile is 90% of COD traffic.
  1. Reproduce the bug: on mobile (or mobile emulator), go to clientsondemand.live/application
  2. On the first question, tap one of the bottom options
  3. Observe: the form container shrinks/collapses and disappears below the fold
  4. Root cause is likely the form container height changing when a selection triggers a transition or conditional field display
  5. Fix: ensure the form container maintains a min-height or uses scroll-into-view behavior so the active step is always visible after selection
  6. Consider adding scrollIntoView({ behavior: 'smooth', block: 'center' }) on the form container after each option selection
/* After each option selection, ensure visibility: */ function onOptionSelect(selectedOption) { // ... existing selection logic ... // Scroll form into view after selection const formContainer = document.querySelector('.form-step.active'); if (formContainer) { setTimeout(() => { formContainer.scrollIntoView({ behavior: 'smooth', block: 'center' }); }, 100); // small delay for DOM update } }
Test on real devices: iPhone Safari and Android Chrome. Emulators don't always catch viewport-resize behavior. Screenshot before/after for each option on step 1.

B2 Hidden GHL Form Integration Add

New B-test application page
Why: Matt's recommendation: "Have the GHL form on the page but hidden and have this newer form populate the form fields. This will make sure we still keep things CRM integrated. If we don't do that we're just going to have to faff around a lot with integrating the form etc."
  1. Keep the existing GHL form embed on the page but hide it with display: none or visibility: hidden; position: absolute; left: -9999px;
  2. The new custom form (our redesigned multi-step flow) collects all the same fields
  3. On final submission of the custom form, programmatically populate the hidden GHL form fields and trigger its submit
  4. This keeps all GHL automations, pipelines, and CRM tagging intact without any backend changes
  5. Map each custom form field to the corresponding GHL form field by name or id attribute
/* Implementation pattern: */ function submitToGHL(formData) { const ghlForm = document.querySelector('.ghl-hidden-form'); // Map our fields to GHL fields ghlForm.querySelector('[name="first_name"]').value = formData.firstName; ghlForm.querySelector('[name="last_name"]').value = formData.lastName; ghlForm.querySelector('[name="email"]').value = formData.email; ghlForm.querySelector('[name="phone"]').value = formData.phone; // ... map all other fields ... // Trigger GHL form submission ghlForm.submit(); }
Field mapping: Before building, inspect the current GHL form on /application and document every field name/ID. The custom form must capture the exact same data points or GHL automations will break.
