For: Rehan  •  From: Kas  •  Date: 2026-04-13  •  Source: Slack therapist LP thread (Matt, Akari, Kas, Russ)
The big picture (ELI5):

We built one good landing page (the therapist LP). It's our master template. We now have other niche pages (educators, coaches, etc.) and they need to match — same video behavior, same footer rules, same CTA style, same copy standards.

Matt reviewed the therapist page and listed a few core things to standardize before we echo the template out to the rest. Your job is to apply that list to every niche LP we have live.

Rule of thumb: Therapist LP = source of truth. If it's different on another niche page, either fix the niche page or flag it to Kas.

Status Update

!Educator Niche LP KAS IS ON IT

Kas is working on the educator LP. You don't need to touch it — but once it goes live, every fix in this guide applies to it too.

Your Fix List (All Niche Pages)

Go through each niche LP we have live (therapist + any others) and do everything below. Therapist first so you have a working reference, then echo to the rest.

1Remove the button animationP1

ELI5: There's a button on the page that has an animation effect (pulse, bounce, or similar). Kill the animation completely. The button should sit still — no movement.

How:

  • Find the button(s) with any animation, transform, or keyframe effect
  • Remove the animation CSS entirely — don't just slow it down
  • Keep the base button style (color, size, hover state) untouched

2Remove "He reviews attendee offers live" copyP0

ELI5: This is NOT about animation — it's a piece of written copy on the page that says something like "He reviews attendee offers live." Delete that line of text. It's been showing up as a question in the workshop FAQ a few times, meaning people are confused by it. Cleaner to just remove it.

How:

  • Search the page HTML for any instance of "reviews attendee offers live" or similar phrasing
  • Delete the line / the bullet / the section containing it
  • Do NOT replace with new copy — just remove cleanly and let the surrounding layout re-flow
  • Repeat on every niche LP (this copy may have been cloned)

Context from Matt: "this has been on a few FAQ instances I think" — attendees asking about it on the workshop, so it's creating friction.

3Footer links (privacy policy etc) open in new tabP1

ELI5: The footer has links like Privacy Policy, Terms, etc. When someone clicks those, they should open in a new browser tab — NOT take the user away from the landing page. We don't want to lose the prospect mid-funnel because they tapped "Privacy Policy" to double-check.

How: Add target="_blank" rel="noopener" to the policy/terms links in the footer. Main nav + CTA links stay as-is.

  • Privacy Policy link
  • Terms of Service link
  • Any other legal / disclaimer footer links

4Check video sound behavior on mobileCHECK ONLY

ELI5: Matt said the video is "playing muted" for him. We've had mobile video problems on COD pages before — Wistia muted autoplay not unmuting properly on mobile HTML. This is NOT a rebuild task — just a CHECK task. Load the page on a real phone, see what the video is actually doing, and report back.

What to check:

  • Load therapist LP on a real iPhone + real Android (not desktop DevTools)
  • Does the video play at all on page load?
  • Is it muted? Can the user tap to unmute easily?
  • Is there a visible play/unmute control?
  • Does the same behavior happen on the other niche LPs?

Report: Screenshot or screen record what you see on mobile and post it in the COD Slack thread. Don't fix anything yet — we need to see the actual state before deciding what to change.

Prior context: We've had bugs where Wistia's muted autoplay-with-scroll-to-unmute doesn't work on mobile HTML builds. If that's what's happening here, Kas will call the fix (likely move to click-to-play).

5Test the $27 payment on mobileP0

ELI5: Actually run through the checkout on your phone and pay the $27 so we know the mobile payment flow isn't broken. After the payment goes through, either refund yourself in the backend OR ping Ryan and ask him to refund it.

How:

  • Open the therapist LP on a real phone
  • Tap the CTA, go through the full checkout flow
  • Complete a real $27 payment
  • Verify: confirmation page loads, confirmation email arrives, booking registers in the backend
  • Refund the charge yourself in the payment backend — OR ping Ryan in Slack and ask him to issue the refund
  • Screenshot any weirdness (broken fields, layout bugs, slow loads, failed redirects)

Why: 90% of COD traffic is mobile. If checkout is broken on mobile, every dollar of ad spend is wasted. Real end-to-end test is the only way to know.

Order of Operations

  1. Start with the therapist LP — it's the source of truth
  2. Apply items 1, 2, 3 on therapist
  3. Run item 4 (video check) on therapist + report back
  4. Run item 5 (payment test) on therapist + refund
  5. Once therapist is clean, echo items 1, 2, 3 to every other niche LP
  6. Re-run video check (item 4) on each other niche
  7. Push staging links to Kas for sign-off before going live

Quality Checklist (before marking done)

  • All button animations removed on every niche page
  • "Reviews attendee offers live" copy removed on every niche page
  • Footer privacy/terms links open in new tab on every niche page
  • Mobile video behavior reported (with screen recording) for every niche page
  • Mobile payment tested + refunded on therapist page
  • Tested on a real phone, not desktop responsive view
  • No console errors
  • Staging links shared with Kas before going live
