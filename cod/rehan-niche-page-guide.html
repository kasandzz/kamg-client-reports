Rehan Guide — COD Niche LP Build & Launch Protocol

Owner: Kas Audience: Rehan Published: 2026-04-14

You're building COD niche landing pages ($27 workshop) for multiple audiences: therapists (live), coaches, attorneys, educators, and more incoming. This guide is the full protocol. Follow it exactly. If something is unclear, ask Kas in DM before you act — do not guess inside GHL.

Source of truth: russ-lp-permanent-rules.md. Read that first, every single time you touch a niche page. This guide sits on top of it.

1. Hard rules — never break

1a. Copy & messaging (FTC + brand)

Akari flagged the following on 2026-04-14. Russ endorsed the catches publicly ("These are all good catches Akari"). They are now permanent rules across every niche LP — educators, coaches, attorneys, and any future niche page.

Never write ✗Write instead / rule ✓
"The River method""The River approach". River is an analogy, not a method. Strip "method" wherever it appears.
"Adapted for educators / attorneys / coaches / [industry]"Remove entirely. The workshop is not adapted for any industry. Niche framing lives in the copy — it is not claimed as a product variant.
"This makes it profitable"Remove. Income claim, not verifiable, FTC risk.
"No new software to master"Remove. Not true.
"Consulting offers""Coaching programs" — matches the actual offer language.
"Private Q&A""LIVE Q&A" — "private" collides with the 1:1 offer mapping.
Income-based hooks for avatars who are not income-motivated (e.g. "Teachers want income and freedom")Lead with the motivation the avatar actually has. For teachers: impact, classroom control, getting their life back. NOT income.

1b. Visual & structure

  • Logo must be the current COD logo on every page. No variants.
  • Footer must be identical across all niche pages: FTC disclaimer block, canonical privacy/terms links, COD address. Copy the footer from the therapist control page — do not rebuild it.
  • "Old way vs New way" box — if the old way is shown first visually, the label on the first block must say "Old way". Match label order to visual order. Do not swap halfway down the page.
  • Hook / hero — every niche page hero must pass Kas's review before it goes live. Do not self-approve niche hero copy.

1c. Mobile-first

90% of COD traffic is mobile. Every change you make gets tested on a real phone (not just the GHL preview) before you mark it done.

1d. FTC pre-pass

Before you hand any niche page to the team for review, run it through the FTC income-claims checker. Save anything the checker misses into a note and ping Kas — those become new permanent rules.

2. Launch communication protocol — never break

Hard rule from Akari 2026-04-14:
"I shouldn't have to ask. If you launched something, you should communicate that before being asked."

Every time you launch, change, or end anything — a split test, a new variant, an ad asset swap, a GHL automation toggle — you post a one-line note in #brickellads-clients-on-demand-pedro.

Launching now: [what] on [which URL]. Test vs [control URL]. Traffic split [X/Y]. No ads-side changes.

Posting the note takes 20 seconds. Not posting it costs the team trust. Non-negotiable.

3. The "delete" rule — never break

You do not use "delete" inside GHL. Ever.

If a split test needs to end: end the test, then create a new variant with a new URL. Do not delete the prior variant. Do not delete the registration page. Do not delete a split test that "looks inactive" — if traffic split is 100% on control, that is a deliberate setting, not a finished test.

Matt Freeman has your back on this — if you are unsure whether ending a test will remove something important, message Matt FIRST in #brickellads or DM. Two-minute check saves an hour of recreation.

Context: on 2026-04-13, registration2 appeared deleted. It wasn't — a new test overwrote it. It cost the team an hour and hit the trust ledger. Don't repeat.

4. Split-test infrastructure

4a. Tracking (PostHog is the goal)

Kas and Matt are moving niche-page split tests to PostHog Option B (anti-flash installation) for accuracy. Until Kas confirms PostHog is live on a given page:

  • Use GHL default split testing
  • Count page views + registrations from GHL + cross-reference Meta Ads Manager
  • Do NOT self-install PostHog or touch Google Tag Manager. Ask Kas or Matt.

4b. URL integrity

Every split test must run on the URL that ads actually drive traffic to. Before you launch a test, check the Meta ad set destination URL. If ads point at clientsondemand.live/registration2 but you set up the test on clientsondemand.live/hto-workshop, the test captures no data.

This happened on 2026-04-13 for coaches — fix in progress.

Pre-launch checklist

  • Ads destination URL confirmed
  • Split-test setup matches ads destination URL
  • Test purchase completed on mobile AND desktop
  • Page passes FTC pre-pass
  • Footer matches canonical
  • Logo is current
  • Akari / Kas have seen the copy
  • One-liner posted in #brickellads
