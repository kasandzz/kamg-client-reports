02What identify() actually does — in 3 sentences

  1. Before identify(): PostHog gives every browser a random anonymous ID (like a temporary cookie). Every event from that browser is tagged with that anonymous ID.
  2. You call posthog.identify('jane@example.com'): PostHog now knows "this anonymous ID is actually jane@example.com." It rewrites all the anonymous events that happened earlier in this session as belonging to jane@example.com, and tags every future event with jane@example.com.
  3. After identify(): Now PostHog video events join cleanly with Hyros sales (both keyed on email). The dashboard query finishes, the Play Rate + Engagement gauges fill in.

04The actual code

Option A — email already in URL or hidden field (simplest)

Drop this in a <script> tag at the bottom of /maf-thanks:

// Wait for PostHog to be loaded (it's already on the page). // Then identify the visitor by email. if (window.posthog) { // Pull email from wherever the form puts it. // Pick whichever ONE applies to this page: const email = new URLSearchParams(window.location.search).get('email') || document.querySelector('#user-email')?.value || window.userEmail || null; if (email && email.includes('@')) { posthog.identify(email.toLowerCase().trim(), { email: email.toLowerCase().trim(), // Optional: any other known traits first_seen_page: window.location.pathname, }); } }

Option B — fire on form blur (for application form)

const emailField = document.querySelector('input[name="email"]'); if (emailField) { emailField.addEventListener('blur', () => { const email = emailField.value.trim().toLowerCase(); if (email && email.includes('@') && window.posthog) { posthog.identify(email, { email }); } }); }

3 things to NOT do

1. Do not call identify() before PostHog is loaded — guard with if (window.posthog).
2. Do not identify() with anything other than email lowercased + trimmed (or the joins will miss).
3. Do not identify() on every keystroke — only on blur or after the email is confirmed.

06What you do NOT have to do

  • Do not change the PostHog initialization code — PostHog is already on the page and capturing events. You're only adding identify() after it loads.
  • Do not change anything in BigQuery, the warehouse, or the dashboard. Once identify() fires correctly, the existing pipeline picks it up automatically.
  • Do not identify() server-side. PostHog identify() is browser-side. Server-side identify is a different API and not what we need here.
  • Do not retroactively backfill old anonymous sessions — PostHog handles that automatically when identify() fires for the first time in a session.
