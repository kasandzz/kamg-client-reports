5. Agent Specifications

Agent 1: Reply Agent P0
Classify cold email replies and draft Claude-powered responses for Julieta's review.

Trigger

Attio webhook when new deal created at "Reply Received" stage.

Flow

Step 1
Read & Classify
Read reply content from Instantly/Email Bison. Classify: interested objection not_interested auto_reply out_of_office
Step 2 (if interested/objection)
Draft Response
Pull company enrichment data from Attio AI Attributes. Draft personalized response using CPG brand voice. Post to Slack #cpg-replies with draft + one-click approve/edit.
Step 3
Send on Approve
On Slack approval: send via original source API. Route by lead_source field (Instantly for Steven leads, Email Bison for Vovik leads). Update Attio: classification, response note, advance to Qualified if interested.
Auto-disqualify
Not Interested / Auto Reply
Auto-move to Disqualified with reason.

Attio fields written: reply_classification, notes (thread context), stage advancement

Dependencies: Instantly API, Email Bison API, Attio SDK, Slack API, Claude API

Agent 2: Sales Intelligence Agent P1
Morning call prep for Krik, pre-call research briefs, auto-generated Gamma proposal decks, and post-call structured data extraction.

Morning Call Prep (Krik Daily Briefing)

Trigger: scheduled, every morning at 8am

  • Check Krik's Calendly/calendar for all calls scheduled today
  • For each call: pull full deal context, company enrichment, ad transparency data, all notes/history
  • Generate a consolidated morning briefing with prep notes for each call
  • Post single Slack DM to Krik: "You have X calls today" + per-call summary cards with key talking points, ICP fit, objection predictions, and recommended approach

Pre-call Brief + Gamma Deck

Trigger: deal moves to Call Scheduled

  • Pull from Attio: company data, deal history, all notes, enrichment
  • Pull similar past deals (same industry/tier) with outcomes
  • Pull ad transparency data from Attio (FB Ad Library + Google Ads fields)
  • Research: website, social presence, additional context
  • Generate brief: overview, ICP fit, suggested CPA range, objection predictions, talking points
  • Auto-generate personalized Gamma deck via Gamma MCP: company-specific slides with their data, industry benchmarks, relevant case studies, proposed pricing tier
  • Post to Slack DM of lead owner (Krik or Juli) with brief + Gamma deck link

Post-call Extraction

Trigger: Attio Pro call recording completed

  • Read call transcript from Attio's native call object
  • Extract: AOV, CPA discussed, objections, next steps, sentiment, timeline
  • Write extracted fields to deal custom fields
  • Auto-advance stage based on call outcome
  • Post summary to Slack #cpg-deals

Dependencies: Attio SDK, Attio MCP Server, Claude API, Slack API, Gamma MCP, Calendar API

Agent 4: Nurture Agent P1
Automatically re-engage cold prospects with context-aware nurture emails. Human-approval required before every send.

Trigger

Scheduled scan (daily). Identifies deals going cold: no activity in >5 days at Qualified, Proposal Sent, or Follow Up stages.

Flow

Step 1
Identify Cold Deals
Query Attio for deals with no activity >5 days. Exclude Disqualified, Closed Won/Lost. Prioritize by lead score + deal value.
Step 2
Build Context Package
For each cold deal, pull: all notes/history, Krik's comments, lead score, qualification status, last conversation topics, objections raised, proposal details if sent, company enrichment data.
Step 3
Draft Nurture Email
Claude generates a personalized follow-up email based on full deal context. Tone matches CPG brand voice. References specific prior conversations. Provides value (case study, insight, resource) rather than just "checking in."
Step 4
Approval Gate (Slack)
Post draft to Slack with deal summary + reasoning. Buttons: Approve & Send Edit Skip Snooze 3 days. Nothing sends without human click.
Step 5
Send & Log
On approval: send via original source API (route by lead_source). Log nurture touch as Attio note. Reset cold-deal timer. Track open/reply if possible.

Nurture Intelligence

  • Cadence: Max 1 nurture email per deal every 5 days. Max 3 nurture attempts before auto-flagging for manual review.
  • Escalation: After 3 unanswered nurtures, notify Krik in Slack DM with recommendation (archive, call, different angle).
  • Learning: Track which nurture emails get replies vs ignored. Feed outcomes back to improve future drafts.

Dependencies: Attio SDK, Instantly API, Email Bison API, Claude API, Slack API

Agent 5: Ad Transparency Checker P1
Validate prospect ad spend claims by checking Facebook Ad Library and Google Ads Transparency Center.

Trigger

Runs on new deal creation (any source) and on-demand before discovery calls.

Flow

Step 1
Resolve Company Identity
Pull company name + domain from Attio. Search for Facebook page and Google advertiser ID.
Step 2
Facebook Ad Library Lookup
Query Meta Ad Library API. Extract: active ad count, ad themes/copy patterns, estimated spend range, creative types, how long running.
Step 3
Google Ads Transparency Lookup
Scrape Google Ads Transparency Center. Extract: active ad count, ad formats, verified advertiser status.
Step 4
Write to Attio
Update Company record: fb_ad_library_status, fb_ad_count, fb_ad_themes, google_ads_status, google_ads_count, ads_verified, ads_last_checked.
ICP Validation

CPG's ICP qualifier is ≥$10K/mo ad spend. Ad transparency data validates whether prospects actually run ads, cross-referenced against their claimed spend. Deals with no active ads get flagged for Krik before discovery call.

Dependencies: Meta Ad Library API, Google Ads Transparency (scraper), Attio SDK

Comments Parsing (Claude-powered extraction)

The Comments column contains rich deal context. The migration script must extract:

  • AOV (e.g., "AOV $85" → aov field)
  • CPA agreed (e.g., "we agreed on a $25 CPA" → cpa_agreed field)
  • CPA target (e.g., "CAC $30-40" → cpa_target field)
  • ROAS (e.g., "1.5 ROAS" → roas_target field)
  • Discount code mentions → discount_code field
  • Objections (e.g., "doesn't want to pay upfront" → tagged)
  • Next steps (e.g., "ping again in February" → follow_up_date)
  • Everything else → deal note

10. Risks & Mitigations

Risk Impact Mitigation
Attio Pro call recording quality < Fireflies Medium Keep Fireflies as backup for 30 days. Evaluate side-by-side.
Team doesn't adopt (again) High "Invisible CRM" strategy: agents do all data entry, team only sees Slack. If Slack adoption fails, the system is broken.
Instantly/Email Bison API changes Medium Abstract sync layer so source changes don't require agent rewrites.
Rate limits hit with multiple agents Low Attio Pro at 100 reads/sec + 25 writes/sec is generous. Implement request queuing as safety net.
Comments parsing misses data Medium Claude-powered extraction with confidence scores. Low-confidence extractions flagged for manual review.
Vovik's Email Bison integration undocumented Medium Park as TBD; Leo investigates API docs during Phase 1. Fallback: manual CSV import.
Gamma MCP reliability for auto-deck generation Low Gamma deck is a bonus, not a blocker. If Gamma API fails, Sales Intel still delivers Slack brief. Queue failed deck for retry.
Nurture Agent over-contacts prospects Medium Hard cap: max 1 email per 5 days, max 3 attempts. Human approval gate prevents any send without explicit click.
