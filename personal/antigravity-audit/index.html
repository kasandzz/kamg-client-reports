Section 1

Antigravity full feature inventory

Cross-referenced from Google's developer blog, Codecademy, Wikipedia, InfoWorld, Hackernoon, and 20+ third-party tutorials. Confidence ratings reflect how many independent sources confirm each feature.

Agent layer

Agent Manager (Mission Control)

Cmd+E · Documented

Spawn up to 5 agents working asynchronously across the same repo or different workspaces, each on a different model. Each agent gets its own conversation thread, plan artifact, and walkthrough.

Footgun: no automatic git worktree isolation. Two agents editing the same file = merge conflicts. Workaround: spawn each in its own worktree before dispatching.

Editor View · Playground · Inbox

Sidebar · Documented

Standard VS Code-style IDE (Editor). Scratch conversation area (Playground) that can be promoted to a real workspace. Cross-conversation tracker (Inbox).

Knowledge Items (KIs)

Auto-generated · Documented

Direct analog to your MEMORY.md. A Knowledge Subagent writes editable markdown facts to ~/.gemini/antigravity/knowledge/<id>/ at conversation end. You can manually edit or delete wrong KIs.

Power-user trick: say "this is experimental, don't create Knowledge Items from this" to suppress storage for a single conversation.

Skills · Rules · Workflows

Filesystem · Documented

The trifecta that mirrors your GSD architecture exactly:

  • Skills — .agent/skills/<name>/SKILL.md with YAML frontmatter description. Loads only when semantic match — solves context bloat. Same shape as Claude Code skills.
  • Rules — AGENTS.md (cross-tool, also read by Cursor) or GEMINI.md (Antigravity-only). Loaded always.
  • Workflows — .agent/workflows/*.md triggered with /name like slash commands. Supports parallel step execution with // parallel annotation (April 2026 update).

Browser / visual layer

Browser View — Chrome extension subagent

Setup gated · Documented

Dedicated browser model that launches Chrome, scrolls, types, clicks, inspects console logs, captures DOM as markdown. Auto-records the session as WebP video stored in browser_recordings/ per workspace.

Default 30s timeout per step — complex flows time out. April 2026 update split form ops into separate inspect-then-submit steps; multi-tab handling fixed.

Browser URL Allowlist

Settings · Documented

Prompt-injection defense — restricts agent navigation to trusted domains. File at ~/.gemini/antigravity/browserAllowlist.txt. Layer Claude Code doesn't have.

Visual Artifacts

Auto · Documented

Screenshots and video recordings of every agent action saved as artifacts. Walkthrough artifact (April 2026) bundles task list + screenshots + video + diff in one shareable view — perfect for client deliverables.

Planning layer

Planning Mode vs Fast Mode

Dropdown · Documented

Planning produces three artifact types before any code runs: Task List (high-level goals), Implementation Plan (verbose technical), Walkthrough (post-execution summary). Inline Google-Docs-style comments — highlight a step, leave a comment, agent revises. Fast mode skips the plan.

Multiple reviewers report Planning Mode is the biggest cost lever — fewer wasted tokens than letting the agent thrash on a bad direction.

Image generation

Nano Banana Pro (native MCP integration)

Auto · Documented

Google DeepMind's image model wired in via MCP. Agent invokes automatically when a task needs visuals — no hotkey. Up to 14 reference images per call for style consistency. Legible in-image text. Frame consistency across multiple generations.

You're not using this if you're not asking for mockups. Try: "generate three hero mockup options for this section."

MCP support

  • Config path: ~/.gemini/antigravity/mcp_config.json — global only, no per-workspace support (officially acknowledged limitation).
  • Critical gotcha: after editing the JSON you MUST click Refresh in the Manage MCP Servers panel. Does NOT hot-reload.
  • Unified Permissions (v1.22.2, Apr 8 2026): Allow/Deny/Ask lists with action(target) syntax — command(prefix), read_file(/path), mcp(server/tool), mcp(server/*).
  • Pre-built integrations: Firebase, Google Cloud Data Cloud (BigQuery, AlloyDB, Spanner, Cloud SQL), Workspace MCPs (Gmail/Drive/Calendar), GitHub, Nano Banana Pro.
  • Recent updates: v1.22.2 unified permissions (Apr 8), workspace tenant support (Apr 2026), Walkthroughs promoted to first-class UI layer.

Hidden / power-user features you're not using

Compiled from Reddit, dev.to, Medium, Hacker News, and the Kerrick Long power-user guide. These are the things even regular users miss.

The filesystem layer Antigravity barely advertises

Antigravity persists almost everything to disk in plain markdown. Most users miss this. On your machine right now:

PathWhat's in it
~/.gemini/antigravity/brain/<conv-id>/Per-conversation task.md, implementation_plan.md, walkthrough.md — grep, version-control, recover via .resolved.N versions
~/.gemini/antigravity/knowledge/<id>/Editable markdown KIs + metadata.json. Fix wrong facts manually.
~/.gemini/antigravity/browser_recordings/WebP video of every browser session, per workspace
~/.gemini/antigravity/html_artifacts/Empty on your machine — first place to look once you start asking for HTML deliverables
~/.gemini/antigravity/conversations/ + implicit/Encrypted protobuf .pb — not user-readable, skip
~/.gemini/antigravity/mcp_config.jsonCurrently has bigquery server bound to green-segment-491604-j8 (COD warehouse)

Turbo annotations — the killer feature nobody mentions

Inside any prompt or workflow file:

  • // turbo — auto-run the single next command without approval
  • // turbo-all — auto-run all subsequent commands in the block

More granular than the global "Always Proceed" setting. The recommended pattern for trusted CI test loops.

The three-file context spine

Power users build a context spine in the repo root instead of pasting long prompts:

  1. AGENTS.md — behavioral constraints, forbidden actions, Definition of Done. Cross-tool standard, also read by Cursor.
  2. GEMINI.md — Antigravity-only override (or global at ~/.gemini/GEMINI.md). Tech requirements, architecture decisions.
  3. brand_guidelines.md — colors, typography, UI tone. Keeps frontend output consistent across iterations.

Workspace-scoped rules also live at .agent/rules/*.md.

The full @ reference set

Most users only know @file and @workspace. The full set:

@workspaceEntire project
@fileSpecific file
@terminalCurrent terminal output — huge for debugging
@problemsErrors + warnings panel
@selectionCurrently selected text
@codebaseIndexed search
@Supabase @Firebase @Cloudflare @Linear @VercelLive MCP-backed queries — the underutilized power move

Keyboard shortcuts most users miss

  • Cmd/Ctrl + L — toggle agent panel then type immediately, never click the chat input
  • Cmd/Ctrl + Shift + L — new conversation thread (cheap, use it more)
  • Cmd/Ctrl + E — flip editor ↔ agent without mouse
  • Cmd/Ctrl + I in terminal — AI terminal command
  • Select text + Cmd/Ctrl + L — pipe terminal output to agent (faster than copy/paste)
  • Sparkle (✨) icon in Source Control — AI commit message generator

Conversation modes & cost levers

  • Planning before Fast — multiple reviewers report Plan-first is the single biggest cost lever. Anything > 30 min should plan first.
  • Model routing per subtask — Gemini 3.1 Pro for planning, Flash for quick edits, Sonnet 4.6 for frontend implementation, Opus 4.6 for heavy reasoning.
  • Manual final QA in embedded browser — automated browser testing burns more tokens than asking the agent to verify. Do final visual check yourself.

Confirmed footguns

  • Cursor settings import broken for some users; onboarding crashes on final step
  • Agent goes unresponsive after failed browser tests — only fix is new conversation thread
  • Doesn't reliably read CLAUDE.md even with long-context claims
  • No queued changes (unlike Claude Code) — can't batch sequential approvals
  • No archive function — only delete, with unclear KI retention
  • Prompt-injection risk — attackers can hide instructions in code comments or docs that exfiltrate via markdown image rendering. Never store API keys in workspace files.
  • Region issues — mainland China needs TUN-mode proxy with US IP

Where it loses vs Cursor / Claude Code

Consensus from XDA, Codecademy, andrew.ooo, Visual Studio Magazine:

  • Cursor wins inline coding flow, polished frontend output, individual dev satisfaction
  • Claude Code wins queued approvals, sequential batching, terminal-native workflow
  • Windsurf wins price ($15/mo) and post-Cognition agent depth
  • Antigravity wins multi-agent orchestration, visual verification via browser recordings, model-per-task flexibility, Google Cloud / BigQuery / Firebase integration

SWE-bench: Antigravity 76.2% vs Cursor ~77% — practically tied.

The critical gap: AGENTS.md doesn't exist in KasOS

Without AGENTS.md at the workspace root, Antigravity's Gemini agent operates with zero context on your rules: GSD conventions, slug routing, FTC compliance gates, em-dash bans, client-isolation rules, Net 0 invoicing, FTC income-claim phrasing. Every time you ask Gemini to do client work, it could violate any of these silently.

One-line fix: symlink AGENTS.md → CLAUDE.md at the workspace root. Antigravity reads AGENTS.md natively. Cursor also reads it. Free portability.

MCP overlap — same backend, separate processes

  • Two independent registries. ~/.gemini/antigravity/mcp_config.json is global only, no per-workspace support.
  • Same MCP server can be registered in both. Each tool launches its own stdio process / holds its own HTTP session. They do NOT share session state.
  • Active risk for your client work: the BigQuery MCP currently bound to COD's green-segment-491604-j8 in Antigravity is visible from every workspace you open, including non-COD client work. Violates your Per-Client Billing Separation rule from MEMORY.md.

The recommended division of labor

Run Antigravity as the shell — the IDE you look at, the diff viewer, the planning artifact UI, the browser-driven visual feedback. Run Claude Code as the engine — the side that knows GSD, all your skills, hooks, plugins, CLAUDE.md, FTC/copy/client-isolation rules.

  • Use the Claude Code panel inside Antigravity for fast edits + slash commands
  • Drop to claude in the integrated terminal for full skill access
  • Reserve Gemini 3 / Jetski for two specific jobs only:
    • Whole-repo planning when you want a different lens on a big architectural change
    • Browser-driven visual iteration on LP rebuilds — point Gemini at the live URL, let its browser subagent screenshot + suggest tweaks, then have Claude Code land the implementation
  • Sync rules via AGENTS.md symlink (one line, one fix)
  • Treat mcp_config.json as the global always-on set (BigQuery, paperclip); ~/.claude/.mcp.json as the rich set (your full kit)
  • Git worktrees whenever you intend to run both agents in parallel — never on the same file
  • Client deliverables with strict isolation (COD, Redstone, CPG): do them in Claude Code only until Antigravity ships per-workspace MCP scope

Action plan, priority-ranked

Ordered by compound leverage. Each estimate is realistic, not aspirational.

Symlink AGENTS.md → CLAUDE.md at KasOS root

~2 min · highest ROI

Gives Antigravity's Gemini agent visibility into every rule you've codified — GSD conventions, em-dash ban, client-isolation, FTC gates, Net 0 invoicing. Without this, Gemini is rule-blind on every prompt. Windows: mklink AGENTS.md CLAUDE.md in cmd as admin. Or just copy the file — sync manually when CLAUDE.md changes.

Install Playwright MCP plugin from marketplace

~15 min · unlocks 3 features in one move

Collapses features 1, 2, 3 from the mapping section. Gives every skill access to browser_navigate, browser_click, browser_snapshot, browser_take_screenshot. 134k installs, vetted official plugin. Path: claude-plugins-official/marketplace.json:1443.

Wire nanobanana-mcp (and update MEMORY.md)

~5 min if API key handy

/ads-generate and /ads-photoshoot are silently broken right now. The skills already assume nanobanana-mcp exists. Add to ~/.claude.json under mcpServers. Then delete or update the stale "nanobanana-mcp configured" claim in MEMORY.md so future sessions stop assuming dormant capability.

Create per-client AGENTS.md files (avoid MCP scope leak)

~20 min · client-safety move

Antigravity's MCP is global — COD's BigQuery MCP is visible to every workspace you open. Until Google ships per-workspace scope, put a per-client AGENTS.md in each kamg-ops/sites/<slug>/ root with explicit "do not query <other client> tables" guard rules. Belt-and-suspenders alongside your existing client-isolation memory.

Port your top 5 GSD workflows as Antigravity workflows

~30 min

Drop markdown files in ~/.gemini/antigravity/workflows/ for: /save, /morning, /compound, /ads create, /lp-audit-rebuild. Antigravity workflows trigger with /name in chat just like GSD slash commands. Add // parallel annotation to independent steps for free wall-clock speedup.

Port high-value skills to Antigravity skills directory

~1 hr · format is nearly identical

SKILL.md format is nearly identical. Start with: lp-audit-rebuild, taste-skill, deslop, theme-factory, ads-create. Drop in ~/.gemini/antigravity/skills/. Antigravity loads only on semantic match via the description field — same mechanic as Claude Code.

Generalize lp-audit's ITERATE band into standalone iterate-on-visual skill

~2 hrs · highest compound leverage

The visual-feedback iteration loop in lp-audit-rebuild/lib/phase-band-runner.js + agents/visual-storyteller.md + agents/code-reviewer.md IS Antigravity's autonomous-iteration feature — just locked to LP rebuilds. Extract as a generic skill: takes any HTML/Astro/JSX file + goal description, runs screenshot → critique → patch → re-screenshot until quality threshold. Add a /sync-rules workflow that regenerates AGENTS.md from CLAUDE.md

~30 min

If symlinks are too brittle on Windows, build a tiny sync workflow: reads CLAUDE.md, writes AGENTS.md, optionally trims sections Gemini doesn't need. Run on Claude Code hook when CLAUDE.md changes.

Add a pre-tool-use guard against same-file races

~45 min

Lightweight hook that refuses Claude Code edits when an Antigravity agent is mid-task on the same file (detect via Antigravity lockfile or .pb brain file modification time). Belt-and-suspenders against the auto-accept compound risk. Optional — only matters if you actually run both agents simultaneously.

Try Gemini 3.1 Pro in Planning Mode for next big architectural decision

free

One specific habit to try: when you have a non-trivial multi-day decision (e.g. PersonalOS/WorkOS split), open it in Antigravity Planning Mode with Gemini 3.1 Pro instead of Claude Code. Compare the plan artifact to what Claude Code's gsd-planner would have produced. If Gemini's bigger effective context gives meaningfully better whole-repo synthesis, you have a new tool in rotation. Skip / already strong: Planning mode, task-level abstraction, multi-file orchestration. You're at or above Antigravity parity on these — don't spend cycles. The compound wins are 1, 2, 3, and 7.
