A four-vector deep dive into the GoHighLevel automation surface: REST API coverage, the Claude Chrome extension, the community MCP landscape, and the funnel object model. Compressed into one decision: where to spend your build time.

TL;DR

You cannot build GHL funnels purely via API. The page builder, forms, and workflows are deliberately not exposed and HighLevel has not budged in 2+ years. The real leverage is Snapshots + Playwright MCP + HighLevel Official MCP, in that order. Plan: clone-and-customize, not build-from-scratch.

Week-One Build Sequence

From zero to first client funnel shipped.

Seven days. The 70% lever first, the customization layer second, the glue third.

DAY 1

Build the master snapshot library

One master snapshot per client vertical (education, fitness, services). One-time UI work, but it's the 70% leverage. Everything downstream rides on this.

DAY 2

Wire HighLevel Official MCP

Add HTTP MCP to ~/.claude.json. Generate scoped PIT in sub-account settings. Test contact + tag + pipeline ops from Claude Code.

DAY 3

Install playwright-mcp + save auth state

Manual login to GHL once. Save auth-state.json to a gitignored path. Smoke test: "open funnel X and rename step 2."

DAY 4

Build /ghl-clone-client slash command

Composes the three layers: (a) snapshot push via API push-status tracking, (b) Playwright-MCP renames pages + swaps copy/images from a client brief, (c) Official-MCP creates client-specific tags + pipeline.

DAY 5

Add n8n-mcp for cross-system glue

Form-webhook → Slack DM → CRM update flows. Import the official HighLevel n8n template (17 ops) as a starting point.

DAY 6-7

Ship one real client funnel end-to-end

Skip Computer Use. Skip Claude Chrome unless you specifically need it for daily admin (Franzi-style ops). Measure: time-to-ship + manual touchpoints + breakage.

What This Means For The Agency

Clone-and-customize, not build-from-scratch.

Stop trying to make Claude "build the page builder."

That's a dead end until HighLevel opens the API, which they have explicitly declined to do for years. Treat funnel production as clone-and-customize:

"1 master snapshot per vertical = your IP. Claude Code + MCPs handle the customization layer. Human owns design pass + OAuth + DNS."

This collapses a 6-8 hour funnel build to roughly 60-90 minutes with current tooling.

Going to zero requires either HighLevel opening the API (out of your control) or building an Astro / Vercel external LP stack and routing GHL forms / webhooks to it. The second path is fully in your control and you already have kamg-astro-starter + 3,008-component library sitting idle. That is the real moat.
