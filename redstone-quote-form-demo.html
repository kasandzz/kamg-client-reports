Section 01

What Changes and Why

Every change below is backed by published CRO research. This is not opinion -- it is measured.

Current Form

  • No progress indicator -- users don't know how many steps remain
  • Gray (#616060) submit button blends into page
  • File upload behind conditional toggle, then "email us instead"
  • No trust signals (ISO, client logos, security)
  • No helper text on technical fields
  • Dedicated form page -- full context switch

Redesigned Form

  • 4-step progress bar with step names + fill indicator
  • Bold Redstone red (#B21E24) primary CTA, green submit
  • Drag-and-drop upload zone with clear format list
  • Trust bar: ISO 9001, IATF 16949, NDA, client logos
  • Inline helper text on every technical field
  • Slideout panel -- form accessible from any page CTA

CRO Evidence Behind Each Fix

Change Research Backing Expected Impact
Progress bar +28% form completion (Zeigarnik effect: incomplete tasks create psychological tension that drives completion) High
High-contrast CTA Primary-color buttons outperform neutral buttons by 20-30%. Von Restorff isolation effect -- the eye goes to the thing that doesn't match High
Trust signals near form +42% conversion when trust badges placed within visual proximity of CTA (not buried in footer) Medium
Helper text Each confusing field = 5-10% abandonment penalty. Expedia removed 1 field and gained $12M/year Medium
Direct file upload Conditional toggles ("do you want to attach?") add a decision point. Every decision = friction. Direct upload zone eliminates the gate Medium
Slideout (vs page) No page exit = no back-button drop-off. Perceived commitment is lower. Tracks which CTA triggered the form High

Slideout Advantages

Benefit Detail
Zero navigation User stays on the page they were reading. No context switch, no back-button drop-off, no "where was I?" moment
Every CTA = same form One form component, triggered from any "Get Quote" button site-wide. Consistent experience, single codebase
Source tracking We log which page and CTA triggered the open -- tells us what content drives quote requests (services page? homepage? specific process page?)
"Takes 2 minutes" anchor Time estimate in the panel header uses anchoring bias -- sets expectation that this is fast, not a big commitment
Mobile: full-screen takeover On mobile (below 768px), the slideout becomes a full-screen modal. Same form, same steps, optimized for thumb reach
Section 04

Implementation Notes

Technical Approach

Component How
Form engine Keep Gravity Forms (already installed). Restyle with custom CSS to match this design. Multi-page pagination already supported
Progress bar Gravity Forms has a built-in progress indicator -- just needs to be enabled and styled with the numbered step design shown above
Slideout panel Custom JS + CSS overlay. Form loads via AJAX inside the panel. Triggered by class on any CTA button site-wide
File upload Gravity Forms advanced file upload field (already supports drag-and-drop). Remove the conditional toggle, make it always visible
Trust bar Static HTML below the form. Uses actual ISO/IATF cert numbers. Client logos are already on the homepage
Tracking PostHog (once approved) tracks: form opens, step completions, drop-off points, time per step, source page/CTA

Field Inventory: Current vs Proposed

Step Fields Avg. Time Cognitive Load
1. Your Project 4 fields (3 dropdowns, 1 card select) ~30 sec Low -- all selection, no typing
2. Specifications 3 fields (1 textarea, 1 upload, 1 text) ~60 sec Medium -- requires thought but is optional-friendly
3. Preferences 2 fields (1 card select, 1 dropdown) ~15 sec Low -- breather step before contact
4. Contact 6 fields (5 text, 1 dropdown) ~45 sec Medium -- autofill handles most of it

Total: 15 fields across 4 steps. ~2.5 minutes. Current form has similar field count but worse distribution and no progress cues.
