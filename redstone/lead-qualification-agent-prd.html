Manuel handles inbound form submissions from redstone-mfg.com. He works approximately 1 hour per day around 5pm. Leads submitted in the morning sit untouched for 5+ hours. Industry data shows manufacturing leads contacted within 5 minutes qualify at ~40%; after 5 hours that drops to ~10%.

Simultaneously, ~70% of inbound leads are unqualified: hobbyists, prototypers, wrong geography, wrong manufacturing process. No one filters these before they reach a human. Eric, Rachel, and Alec all lose time on dead leads.

2. Objective

Build an AI agent that lives inside HubSpot (where leads already land via WPForms) that:

  1. Scores and qualifies every inbound lead against Redstone's ICP within minutes of submission
  2. Routes qualified leads to the right person with full context
  3. Filters junk so humans only touch leads worth their time
  4. Logs everything so Eric has a table of every lead, score, and outcome
  5. (Future) Sends an instant acknowledgment to the prospect

3. Ideal Customer Profile (Qualification Criteria)

The agent scores leads against these dimensions, derived from Eric's historical deal patterns:

Dimension Qualified (Score +) Disqualified (Score −) Weight
Manufacturing Process Die casting, CNC machining, injection molding, stamping, forging 3D printing, jewelry, crafts, "how to" queries High
Volume / Scale Production runs, OEM, ongoing supply Single prototype, hobby project, 1-off personal item High
Geography North America, Europe (companies sourcing from MX/CN) Regions Redstone doesn't serve; local consumer requests Medium
Company Signal Has company name, company email, identifiable business Gmail/Yahoo personal email, no company info, student Medium
Deal Size Indicator Mentions quantities, RFQ, annual volume, existing supplier Asks about single unit pricing, "just curious" Medium
Urgency Timeline mentioned, active project, switching suppliers Research phase, no timeline, academic inquiry Low

These weights and criteria are configurable. Phase 2 tunes them using historical HubSpot deal data (won vs. lost).

4. System Architecture

Design Principles

  • HubSpot-native: Leads already flow to HubSpot via WPForms. The agent triggers from HubSpot, writes back to HubSpot. No new systems for Eric to learn.
  • Cloud-hosted: Runs on a lightweight cloud function (GCP Cloud Run or AWS Lambda) so it's always on, not dependent on anyone's laptop.
  • Handover-ready: Eric's team can own this. Config lives in a simple JSON/spreadsheet. No black box.
  • POC first: Eric's rule: "Confirm it works manually, then automate." Phase 1 runs in shadow mode alongside Manuel.

Data Flow

WPForms (redstone-mfg.com) ↓ auto-sync HubSpot Contact created (existing flow, no change) ↓ webhook fires on new contact Cloud Function receives payload ↓ LLM Qualification Engine • Reads: form fields, company name, email domain, message text • Enriches: company lookup (Clearbit/Apollo, optional) • Scores: ICP match (0-100), qualification tier, reasoning ↓ Writes back to HubSpot • Custom properties: lead_score, qualification_tier, agent_reasoning • Assigns contact owner based on routing rules • Moves to appropriate pipeline stage ↓ Notification to assigned human (HubSpot native or Slack) ↓ (Phase 5, future) Instant email to prospect acknowledging receipt

5. Phased Roadmap

1 Shadow Mode MVP

Goal: Prove the agent qualifies leads as well as (or better than) a human, without touching the live workflow.

TaskDetail
HubSpot custom propertiesCreate: lead_score (number 0-100), qualification_tier (Hot / Warm / Cold / Junk), agent_reasoning (text), agent_processed_at (datetime)
Webhook setupHubSpot workflow: on new contact creation → POST to Cloud Run endpoint
Cloud functionReceives contact payload, calls Claude Haiku with ICP prompt + form data, writes score back to HubSpot via API
ICP prompt engineeringSystem prompt encoding the qualification table from Section 3. Structured JSON output: score, tier, reasoning, suggested_owner
Shadow loggingAgent writes scores but does NOT change contact owner or pipeline stage. Manuel continues as-is.
Validation dashboardSimple HubSpot report: agent score vs. Manuel's actual disposition. Weekly review table for Eric.

Success criteria: Agent agrees with Manuel's qualification on ≥80% of leads over 2-week shadow period. Eric reviews the comparison table and approves go-live.

Estimated effort: 1-2 weeks

Cost: ~$0 incremental (HubSpot free tier supports webhooks; Cloud Run free tier; Haiku costs ~$0.001/lead at 50 leads/mo)

2 Live Qualification + Routing

Goal: Agent takes over qualification. Qualified leads auto-route to the right human. Manuel stops triaging.

TaskDetail
Auto-assign contact ownerRouting rules: Hot → Eric/Rachel, Warm → Alec, Cold → nurture queue, Junk → archived
HubSpot task creationHot leads create a HubSpot task for the assigned owner with agent reasoning + next-step suggestion
NotificationSlack webhook or HubSpot email notification on Hot leads (configurable)
Routing configGoogle Sheet or JSON: who gets which tier, business hours, fallback owner
Override mechanismAny human can re-score a lead in HubSpot; agent logs the override for model tuning

Success criteria: Response time to Hot leads drops below 30 minutes. Eric confirms junk leads no longer reach humans.

Estimated effort: 1 week (incremental on Phase 1)

7. Risks & Mitigations

RiskSeverityMitigation
Agent misqualifies a high-value lead as junk High Phase 1 shadow mode catches this before go-live. Conservative threshold: anything uncertain goes to Warm (human review), not Junk. Weekly audit table for Eric.
HubSpot API limits Low Redstone volume is ~50 leads/month. HubSpot free tier allows 100 API calls/10 sec. Not a concern.
Eric doesn't trust AI scoring Medium Shadow mode comparison table is the trust-builder. Show him the data (table, not chart). His quote suggests he's already bought in conceptually.
Manuel's role during transition Medium Phase 1 = shadow (Manuel keeps working). Phase 2 = agent takes over qualification; Manuel can shift to follow-up/relationship tasks. Eric decides Manuel's role.
LLM cost creep Low Haiku at ~$0.001/lead. 50 leads/month = $0.05/month. Even 10x growth is negligible.
Form data insufficient for qualification Medium Phase 1 validates this. If form fields are too sparse, recommend adding 1-2 qualifying questions to the form (Eric's existing "form trim" initiative).
