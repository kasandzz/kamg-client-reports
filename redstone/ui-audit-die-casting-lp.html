S

Strengths

S-01 Excellent

Hero Section Information Architecture

The hero combines a strong value proposition with three quantified proof points (24-48hr quote, 8-12wk production, +/-0.015" tolerance) in a .hero-meta-row immediately below the CTAs. This is textbook B2B lead-gen architecture: headline > subhead > dual CTA > social proof stats. The left-aligned layout avoids the generic centered hero pattern.
Lines 1612-1638 - hero-inner grid, hero-meta-row with three data points. CSS L94-200 - responsive scaling with clamp() for headline.
S-02 Excellent

Design System Fidelity

The page uses Redstone's design tokens consistently throughout: --rs-red exclusively for CTAs (never for card/section fills), --rs-ink for dark sections, --rs-line for dividers, and the full type scale from the colors_and_type.css file. Square corners (border-radius: 0) are maintained globally, consistent with the Figma system. The 1px gray dividers create a clean engineering-document aesthetic.
CSS L167-168 - button border-radius: 0. All color values reference CSS custom properties, zero hardcoded hex values for brand colors.
S-03 Strong

Trust Signal Layering

Trust is built progressively: header phone number > hero stats > trust strip (IATF 16949, ISO 9001, U.S. Managed, 8 Years) > client logos (MIT, Amazon, Honda, Halliburton) > qualification checklist > FAQ > final CTA trust rail. The form itself includes a "Engineers are online" live indicator with green pulse dot, certification badges, and a 24-48hr quote badge. This multi-layer approach is high-conversion for B2B manufacturing.
Lines 1641-1664 - trust strip. Lines 2113-2150 - form header with trust signals. Lines 2275-2288 - final CTA trust rail.
S-04 Strong

Multi-Step Form with Progress Indicator

The 3-step form (Parts > Specs > Contact) reduces perceived complexity. The blueprint grid background treatment and corner-tick decorations differentiate it from generic forms. Step validation is implemented (L2427-2467) with field-level error states. The form glow animation on CTA click (L477-485) provides clear visual feedback that draws attention to the form.
CSS L1061-1370 - form blueprint treatment. JS L2410-2467 - step navigation and validation logic.
S-05 Strong

Sticky Right-Rail Form Pattern

The .form-col uses position: sticky; top: 96px; to keep the quote form visible as users scroll through content. This is the highest-converting layout for B2B lead gen pages, keeping the conversion action always within reach. At breakpoints below 1080px, it correctly unsticks and stacks below content.
CSS L316-337 - twocol grid with sticky form column. 440px fixed form width with responsive fallback.
S-06 Good

Scroll-Driven Micro-Interactions

Process steps use staggered IntersectionObserver reveals (L2582-2594) with sequential transition-delays. Section titles get a highlight sweep animation (L736-746). CTA buttons have a subtle glow pulse (cta-nudge keyframes). The unit toggle (Imperial/Metric) for capability stats is a thoughtful B2B UX detail. The next-field nudge animation (L2530-2554) guides form completion. All animations respect prefers-reduced-motion (L1576-1579).
CSS L450-476 - CTA nudge + arrow bounce. CSS L1576-1579 - reduced motion support.
S-07 Good

Semantic HTML and Accessibility Foundations

Skip-to-content link present (L1584), semantic landmarks (header, main, footer, section, nav with aria-label), FAQ buttons with aria-expanded, unit toggle with role="switch" and aria-checked. Logo carousel correctly uses aria-hidden="true" on duplicate items. The .visually-hidden class is properly implemented.
Line 1584 - skip link. Line 1592 - nav aria-label. Line 1711 - unit switch aria attributes.
S-08 Good

Qualification Checklist as Interactive Pre-Qualifier

The interactive checklist (L2029-2054) lets prospects self-qualify before submitting. Checking items triggers contextual messaging ("Strong fit" / "Likely fit"). This reduces unqualified leads and increases conversion quality, which is critical for a $50K+ minimum program. The checklist items match the ICP criteria exactly.
Lines 2029-2062 - qualification section with 4 checklist items. JS L2389-2408 - dynamic result messaging.
W

Weaknesses

W-01 High

Mobile Form UX: Sticky Form Breaks on Tablet/Mobile

Below 1080px the form unsticks and sits below all content sections (L334-337). On mobile, users must scroll through the entire page to reach the form. The mobile bottom bar (L2326) partially addresses this, but it only shows after the form leaves the viewport. The form itself is 440px wide with dense fields, which on a 375px viewport creates either horizontal overflow or extremely compressed inputs.
CSS L320 - grid-template-columns: minmax(0, 1fr) 440px. CSS L334-337 - mobile fallback to single column.
Recommended Fix
Add a prominent "Get a Quote" floating CTA on mobile that opens the form in a slide-up modal/drawer. Reduce form width to fluid on mobile. Consider showing Step 1 inline on mobile and Steps 2-3 in a modal.
W-02 High

No Step 2 Validation - Users Can Skip Required Fields

Step 1 has validateStep1() and Step 3 has validateStep3(), but Step 2 has no validation at all (L2452-2457). The "Continue" button on Step 2 proceeds to Step 3 without checking required fields (annual-spend, part-description, country-production are all marked with * but never validated). This means submissions can arrive with empty critical fields.
JS L2452-2457 - data-next click handler, no validateStep2() function exists. Lines 2178-2206 - Step 2 fields with * required markers.
Recommended Fix
Add a validateStep2() function that checks annual-spend (non-empty select), part-description (non-empty textarea), and country-production (non-empty input). Wire it into the data-next handler at line 2454.
W-03 High

Form Has No Backend Submission

The form uses onsubmit="return false;" (L2111) and the submit button simply hides steps and shows a success message (L2462-2467). No data is sent anywhere. No fetch/XHR call, no form action, no hidden iframe. The success message tells users to "check your inbox for a confirmation" which will never arrive. This is a conversion-killing issue if the page goes live as-is.
Line 2111 - onsubmit="return false;". JS L2462-2467 - submit handler only shows success UI.
Recommended Fix
Integrate form submission via fetch() to the appropriate endpoint (Redstone's CRM, a Zapier webhook, or a serverless function). Add a loading state to the submit button. Handle error states with retry UI.
W-04 Medium

Exit Intent Modal Accessibility and Mobile Issues

The exit modal (L2334-2351) uses inline styles exclusively, has no focus trap, no ESC key handler, and the close button is a bare × character with no aria-label. The modal only triggers on mouseout which means it never fires on mobile/touch devices. The modal's "Send Feasibility Check" button does nothing (inline onclick just hides the modal without submitting data).
Lines 2334-2351 - exit modal HTML with inline styles. JS L2517-2528 - mouseout trigger only.
Recommended Fix
Extract inline styles to CSS classes. Add focus trap (trap focus inside modal when open). Add ESC key listener. Add aria-label to close button. Consider a scroll-depth exit intent for mobile (trigger at 80% scroll-up velocity). Wire the submit button to actual form submission.
W-05 Medium

Logo Carousel Animation Performance

The logos use a CSS @keyframes logo-scroll with an 80s linear infinite animation (L290-293). The entire .logos-grid container with all images animates via transform. While transform is GPU-accelerated, the container holds 14 images (7 original + 7 duplicates). There is no will-change: transform hint, and images lack explicit width/height attributes, which can cause CLS (Cumulative Layout Shift) during load.
CSS L278-293 - logos-viewport mask + scroll animation. Lines 1674-1689 - 14 img elements in carousel.
Recommended Fix
Add will-change: transform to .logos-grid. Add explicit width/height attributes to all logo images to prevent CLS. Consider pausing the animation when the section is not in the viewport to save GPU cycles.
W-06 Medium

Font Loading Chain Creates Render Delay

The page loads Host Grotesk from Google Fonts (L18) AND ZT Nature from local font files via colors_and_type.css (18 @font-face declarations for weights 100-900). The CSS uses font-display: swap which is good, but loading 18 font files is excessive. The page only uses weights 300, 400, 500, and 700 of ZT Nature. The remaining 14 font files (100, 200, 600, 800, 900 + all italics) are dead weight.
CSS (colors_and_type.css) L8-43 - 18 @font-face declarations. Line 18 - Host Grotesk Google Fonts link.
Recommended Fix
Subset ZT Nature font files to only include weights actually used (300, 400, 500, 700 normal only - no italics used on this page). This reduces font file loads from 18 to 4. Remove the Host Grotesk Google Fonts link if ZT Nature is the primary typeface.
W-07 Medium

Em Dash Characters in Copy

Multiple instances of &mdash; appear in the page copy: "your engineers speak to the production floor through one program manager" (L1810), the exit modal "Before you go" (L2339), and the file upload note (L2199). While typographically valid, these create inconsistency as some other separators use &ndash; or &middot; instead.
Lines 1810, 2199, 2339 - em dash usage. Lines 1720, 1738, 2074 - en dash usage for ranges.
Recommended Fix
Standardize punctuation: use en dashes for ranges (correct already), middle dots for separators, and regular dashes or commas where em dashes currently appear.
W-08 Low

Process Section Dark Background Inside Light Content Column

The #process section (L692-800) uses background: var(--rs-ink) (dark) inside the left content column, which itself sits inside a light white page. This creates an awkward visual where a dark-background section has its right edge cut off by the sticky form column. On desktop at 1440px, the dark process block occupies only about 60% of viewport width.
CSS L692-698 - process section with dark bg and margins. CSS L316-324 - twocol grid constraining the process section.
Recommended Fix
Either make the process section full-width (break out of the twocol grid) so the dark background spans edge-to-edge, or switch it to a light background with dark text to match the surrounding content column. The current hybrid breaks the visual flow.
W-09 Low

Qualification Section Missing Border Context

The #qualification section (L972-976) has padding: 96px 0 and border-top: 1px solid var(--rs-line), but it is positioned inside the content column where the other sections use .section class for consistent spacing. The qualification section uses a different pattern (.qual-inner with its own background and border) creating visual inconsistency in the section rhythm.
CSS L971-981 - qualification section styling vs. CSS L342-373 - standard section styling.
Recommended Fix
This is intentional as a standalone callout. Minor issue. Ensure the 96px top padding matches the section rhythm. Consider adding the .section class for consistent spacing.
O

Opportunities

O-01 High Impact

Add a Testimonial / Case Study Section

The page has zero customer testimonials, case studies, or quantified results. For a B2B manufacturing page targeting $50K+ programs, social proof from named clients (especially those whose logos are shown: Honda, Halliburton, Amazon) is critical. The logo carousel proves they work with big names, but without a single quote or case study, the credibility stops at "we worked with them."
No testimonial section exists in the HTML. Logos section (L1667-1692) shows 7 client logos without any supporting narrative.
Recommended Fix
Add 2-3 customer testimonials between the Industries and Qualification sections. Format: named quote with company, title, and a specific metric ("Redstone reduced our per-unit cost by 32% while maintaining IATF tolerance specs"). Even one strong testimonial from a recognizable company would significantly boost conversion.
O-02 High Impact

Add Live Chat or WhatsApp Widget for International Buyers

The page targets "Mexico operations" and lists facilities in Mexico, Brazil, China, India, and Taiwan on the map. International buyers may not want to fill out a multi-step form or call a U.S. phone number. There is no live chat, WhatsApp button, or other low-friction contact option for non-U.S. prospects.
Lines 2267-2273 - only contact option is U.S. phone and form. Lines 882-888 - map showing global locations.
Recommended Fix
Add a floating chat widget (Intercom, Drift, or WhatsApp Business) positioned at bottom-right. On mobile, integrate into the mobile bottom bar as a third option alongside Call and Quote.
O-03 High Impact

Add Urgency/Scarcity to Quote CTA

The page has 8 separate "Request a Quote" CTAs but none include any urgency mechanism. No capacity indicator, no limited availability messaging, no seasonal pricing context. B2B manufacturing buyers are used to long lead times, so any urgency signal (even mild) differentiates.
All CTA elements: hero (L1619), capabilities footer (L1797), mid-CTAs (L1834, L1918), qualification (L2059), final CTA (L2260-2261).
Recommended Fix
Add a subtle capacity indicator near the form: "Current capacity: accepting new programs for Q3 2026" or "3 tooling slots remaining this quarter." This creates legitimate urgency without being pushy. Update the form badge to include a time-sensitive element.
O-04 Medium Impact

Map Section Has No CTA

The map section (L802-894) is a full-width dark section showing global supply chain locations, but it has no CTA. It is a dead-end visually. Users see the map, then scroll to the next section. This is wasted real estate - the map demonstrates global capability which should feed directly into conversion.
Lines 802-894 - map section with only a title and map graphic, no CTA or action item.
Recommended Fix
Add a CTA below the map: "Consolidate your global supply chain. Request a quote." with a button linking to #quote-form. The map proves geographic reach, which is a natural lead-in to a reshoring/nearshoring conversation.
O-05 Medium Impact

Add Part Gallery or Portfolio Section

No photos of actual die-cast parts appear on the page. The hero uses a molten pour image (referenced as assets/hero-pour.jpg), but there is no gallery of finished parts. Engineering buyers want to see the quality of work, surface finish quality, and part complexity before reaching out. The process section uses placeholder SVG sketches instead of real photos.
Lines 1935-2017 - process steps with SVG placeholder sketches. Line 108 - hero background image reference.
Recommended Fix
Add a "Recent Work" or "Part Gallery" section between Capabilities and Why Redstone. Show 6-8 photos of actual die-cast parts with material, finish, tolerance, and industry tags. This is extremely high-conversion for manufacturing pages.
O-06 Medium Impact

Form Step 1 Should Auto-Select "Die Casting"

The "Type of Manufacturing Process" dropdown in Step 1 (L2143-2154) already has selected on "Die Casting" which is correct. However, "Current Stage of Production" (L2135) defaults to a blank placeholder. Since this is a die casting-specific landing page, the first question should default to the most common stage or be replaced with something more qualifying.
Lines 2135-2140 - current stage select with empty default. Lines 2143-2154 - manufacturing process with die casting pre-selected.
Recommended Fix
Consider restructuring Step 1 to lead with the "Main Reason" checkboxes (which is the most qualifying question) and move the stage/process selects to Step 2. The first interaction should feel easy and relevant.
O-07 Low Impact

Add Schema.org Structured Data

The page has good meta tags (OG, Twitter card) but no JSON-LD structured data. Adding Organization, Service, and FAQ schema would improve search visibility and enable rich results for the FAQ section.
Lines 2-14 - existing meta tags. No <script type="application/ld+json"> present.
Recommended Fix
Add JSON-LD for: Organization (Redstone Manufacturing, Ohio), Service (Die Casting Services), and FAQPage (the 5 FAQ items). This is a 15-minute implementation with significant SEO upside.
T

Threats

T-01 Critical

Exit Modal and Form Submit Are Non-Functional

If this page goes live, two conversion endpoints are completely broken: (1) The main form shows "Thank you" but sends no data anywhere (L2462-2467). (2) The exit intent modal's "Send Feasibility Check" button just closes the modal (L2349) without capturing the email or part description. Both create a worst-case scenario: the user believes they have submitted, but Redstone never receives the lead. This would directly lose revenue.
Line 2111 - form onsubmit="return false;". Line 2349 - exit modal onclick closes without submission.
Recommended Fix
P0 - before going live: wire both endpoints to a backend (GHL webhook, Zapier, or direct API). Add loading state, error handling, and email confirmation trigger. This is the single highest-priority fix.
T-02 High

Asset Dependencies May Break on Production Domain

The page references local assets with relative paths: assets/hero-pour.jpg, assets/redstone-logo.svg, assets/client-logo-*.png, and colors_and_type.css?v=zt3. The CSS references fonts at fonts/ZTNature-*.woff2. If the page is deployed to any path other than the root of the assets directory, all images and fonts will 404. The OG image also uses a relative path (L13).
Line 13 - relative OG image. Line 19 - relative CSS link. Lines 1675-1688 - relative logo image paths.
Recommended Fix
Convert all asset paths to absolute URLs or ensure the deployment structure matches. The OG image must be an absolute URL for social sharing to work. Test all font files load correctly from the production domain.
T-03 Medium

Mobile Navigation Hidden with No Hamburger Menu

At breakpoints below 900px, .header-nav { display: none; } (L87) hides the navigation links entirely. There is no hamburger menu, no mobile menu drawer, no alternative navigation. Users on tablet/mobile lose access to in-page navigation (Capabilities, Why Redstone, Process, Facilities, FAQ). The only remaining header elements are the phone number and "Get a Quote" button.
CSS L85-91 - media query hiding nav. No hamburger button or mobile menu exists in the HTML.
Recommended Fix
Add a hamburger menu button that opens a full-screen or slide-in navigation drawer on mobile. Include the same anchor links plus the phone number and CTA button. This is essential for usability on mobile.
T-04 Medium

No Tracking or Analytics Integration

The page has zero analytics: no Google Analytics/GA4, no Google Tag Manager, no Meta Pixel, no PostHog, no conversion tracking of any kind. Form submissions, CTA clicks, scroll depth, FAQ interactions, qualification checklist engagement, and exit intent triggers are all untracked. Without analytics, there is no way to measure conversion rate or optimize the page.
No analytics script exists in the page. No data-layer events, no tracking pixels, no conversion tags.
Recommended Fix
Add GTM container with: GA4 pageview, form_submit event on each step completion, quote_submitted conversion event, scroll_depth tracking, and CTA click events. If running paid traffic, add Google Ads conversion tag and Meta Pixel if applicable.
T-05 Medium

Color Contrast Issues in Secondary Text on Dark Backgrounds

Several text elements on dark backgrounds fail WCAG AA contrast requirements: rgba(255,255,255,0.5) on var(--rs-ink) (#262626) in process numbers (L749) yields approximately 4.0:1, below the 4.5:1 threshold for normal text. The footer tagline at rgba(255,255,255,0.5) on dark bg is similarly low. The hero sub text at rgba(255,255,255,0.78) passes, but the hero meta labels at rgba(255,255,255,0.65) are borderline.
CSS L749 - process-num color: rgba(255,255,255,0.5). CSS L1549 - footer-tagline rgba(255,255,255,0.5). CSS L192-193 - hero-meta-l rgba(255,255,255,0.65).
Recommended Fix
Increase minimum opacity for white text on dark backgrounds to 0.72 for body text and 0.55 for large text. This brings all text to WCAG AA compliance without significantly altering the visual hierarchy.
T-06 Medium

CTA Glow Animation May Appear Spammy

The cta-nudge animation (L450-459) applies a pulsing red glow to ALL anchor links pointing to #quote-form with class .btn-primary, .btn-onward, or .finalcta-cta. This means 5+ buttons on the page are simultaneously pulsing with a red glow every 3 seconds. Combined with the arrow bounce animation on text-style CTAs (L468-475), the page can feel aggressive. This may undermine the premium, engineering-focused brand positioning.
CSS L449-476 - CTA nudge and arrow bounce keyframes. CSS L454-459 - selector targeting all quote-form anchors.
Recommended Fix
Limit the glow animation to only the hero CTA and the final CTA. Remove it from mid-page CTAs. Alternatively, trigger the glow only once when the CTA scrolls into view (via IntersectionObserver) rather than running infinitely.
T-07 Low

Map Section May Not Load Correctly

The map section references an image via .map-stage img but no <img> tag was found in the map section HTML. The map might be implemented elsewhere or may be incomplete. The callout labels (.map-callout) are positioned with absolute percentages relative to a container that may have no visible content, resulting in floating labels over nothing.
CSS L827-851 - map-stage with aspect-ratio and img styling. Lines 802-894 - section exists in CSS but map HTML was not found in the content column.
Recommended Fix
Verify the map section HTML is present and the map image asset exists. If the map section is not ready, remove the CSS to avoid dead code. The map section sits outside the twocol grid (it appears after #main-region in the CSS but its HTML placement needs verification).

Executive Summary

This is a well-architected B2B landing page with strong design system adherence, excellent trust signal layering, and a smart sticky form pattern. The information architecture follows proven B2B lead-gen conventions, and the engineering-themed visual treatment (blueprint grid, corner ticks, monospaced labels) differentiates it from generic manufacturing pages.


However, three critical issues must be resolved before launch: the form submits no data, the exit modal captures nothing, and there is zero analytics tracking. These are not design issues, they are functional blockers that would result in lost leads and zero visibility into page performance.


The highest-ROI improvements beyond the critical fixes are:

  • P0 - Wire form and exit modal to backend (Redstone CRM, GHL, or webhook). Without this, no conversion can occur.
  • P0 - Add analytics tracking (GTM + GA4 + conversion events). Cannot optimize what you cannot measure.
  • P1 - Add 2-3 customer testimonials with named clients and specific metrics. This is the single highest-conversion content addition for B2B.
  • P1 - Fix mobile navigation (add hamburger menu) and mobile form UX (slide-up modal or inline Step 1).
  • P2 - Add part gallery section showing finished work. Engineers want visual proof before reaching out.
  • P2 - Add Step 2 form validation and fix contrast ratios on dark background text.
  • P3 - Reduce font file loads from 18 to 4, add structured data, limit CTA glow animations.
